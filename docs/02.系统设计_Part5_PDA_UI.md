# 七鑫智能物料架系统 · Android PDA 端 UI/UX 设计

> 读者对象：前端开发、UI/UX 设计师、QA
> 设计原则：大字大图、抗疲劳、单手操作、扫码即操作、离线可用

---

## 1. 终端环境

| 项目 | 规格 |
|------|------|
| 设备 | Android PDA / 安卓手机 (7-10 寸) |
| 操作系统 | Android 9+ |
| 框架 | React Native (Expo) |
| 运行模式 | Kiosk 全屏 (禁用返回键/通知栏) |
| 主要交互 | 扫码 (PDA 内置扫码头) + 触摸屏 |
| 网络 | WiFi 局域网 + 断网缓存 |

### 3.2 抗疲劳设计原则

| 原则 | 实现 |
|------|------|
| 大按钮 | 最小点击区域 48×48dp |
| 大字体 | 正文 18sp 起步 |
| 高对比度 | 遵循 WCAG AA (4.5:1) |
| 减少输入 | 扫码代替键盘输入 |
| 防误触 | 删除/取消操作二次确认 |
| 视觉反馈 | 扫码成功 绿闪 + 震动 + 蜂鸣 |

## 5. 交互细节

### 5.1 扫码反馈

| 场景 | 视觉 | 听觉 | 触觉 |
|------|------|------|------|
| 扫码成功 | 绿屏闪光 100ms | 短蜂鸣 ×1 | 轻微震动 |
| 扫码失败 | 红屏闪光 200ms | 长蜂鸣 ×2 | 强烈震动 |
| 防呆拦截 | 黄屏 + 警告弹窗 | 三短蜂鸣 | 持续震动 500ms |
| 出库确认 | 绿屏 + ✅ 动画 | 两声蜂鸣 | 两次震动 |

### 5.2 亮灯实时同步

- 使用 WebSocket 推送亮灯状态变化
- 离线时: 轮询 API (10s间隔)
- 亮灯颜色:
  - 🟢 绿色 = 取料指引
  - 🟡 黄色 = 注意/尾数提示
  - 🔴 红色 = 紧急/补料

## 6. PDA 端技术栈

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 框架 | React Native (Expo) | 跨平台, 热更新 |
| 路由 | expo-router | 文件路由 |
| 状态管理 | Zustand | 轻量, 全局状态 |
| 网络 | axios + axios-retry | HTTP 请求, 自动重试 |
| 离线缓存 | SQLite (expo-sqlite) | 断网数据缓存 |
| 扫码 | ZXing (com.google.zxing:core) | 1D/2D 条码解码, 替代 ML Kit |
| WebSocket | react-native-websocket | 实时亮灯同步 |
| 本地通知 | expo-notifications | 超时未出库提醒 |
| 图表 | react-native-chart-kit | 报表展示 |
| 文件导出 | expo-file-system | Excel 导出 |
