### 3.1 API 架构总览

```

    │  Base URL: https://192.168.1.XX:8080/api/v1

    │
    ▼
┌──────────────────────────────────────────────┐
│              FastAPI Application               │
│                                              │
│  ┌────────────┐  ┌────────────┐  ┌─────────┐│
│  │ Auth Router │  │ Material   │  │ Receipt ││

│  └────────────┘  └────────────┘  └─────────┘│
│  ┌────────────┐  ┌────────────┐  ┌─────────┐│
```

### 3.8 XR 点料机对接

> XR 点料机型号: XC1000
> 对接方式: XR 设备主动 POST 上报到 MES API
> 上报地址: `http://<mes-ip>:8080/api/v1/xr/upload`
> 业务流程: 放料盘 → 读码 → 点数 → 更新条码数量 → 提交料架 → 打印新标签

#### POST /api/v1/xr/upload — XR 数据上报

```json
// Request (XR 点料机主动 POST)
{
    "reelID": "SMT-REPAIR01",     // 料盘唯一码 (从条码解析: 物料编码 + 批次)
    "Qty": 5000                     // 点数数量
}

// Response
{
    "success": true,
    "code": 0,
    "message": "OK:上传成功"
}

// 失败响应
{
    "success": false,
    "code": -1,
    "message": "物料不存在 / 配对失败 / 重复提交"
}
```

#### XR 标签解析规则

```
原始条码格式: 4500067189 NI000035181600 030000010
                      ↑               ↑
                   物料编码          数量+序列号

解析逻辑:
  1. 提取物料编码: 4500067189 (前10位)
  2. 提取批次号: NI000035
  3. 提取数量: 181600 (中间6位) → 1816.00 (换算)
  4. 序列号: 030000010 (末尾)

点数后新标签: 4500067189 NI000035 181600029990010
                ↑物料   ↑批次  ↑新数量2999 + 序列号

更新流程:
  XR 上报 Qty=5000 → MES 解析 reelID → 找到匹配库存盘 →
  更新 inventory_pallets.quantity → 标记 ready_restock →
  生成新标签: 物料编码 + 批次 + 新数量 + 序列号
```
