1. 功能概述
灯测试（Light Test）是智能料架硬件诊断功能，用于向物理料架发送测试指令，驱动其所有 LED 灯珠执行一系列预设的灯光模式，从而验证硬件完整性。属于独立于生产流程之外的运维/调试工具。
2. 支持的测试模式

|模式	|值(bitmask)	|说明|
|---|---|---|
|取消测试	|0	|停止正在运行的测试|
|RGB 灯珠测试	|1	|逐个测试 RGB 三色灯珠（色彩循环）|
|灯序测试	|2	|测试灯光顺序（逐个点亮/熄灭）|
|警示灯测试	|4	|测试警示/指示灯|
|感应传感器测试	|8	|测试接近/感应传感器|
|全部测试（默认）	|15	|1 + 2 + 4 + 8 = 全功能诊断|
默认调用 test_mode=15，执行全面硬件检查。
3. 用户操作流程
┌─────────────────────────────────────────────────────────────────────┐
│  PC Web 浏览器（管理员 / 硬件工程师）                               │
│                                                                     │
│  Step 1: 导航到 系统管理 → 料架管理 (/shelf)                       │
│          → ShelfManagementPage.tsx                                  │
│                                                                     │
│  Step 2: 查看料架列表 (编号、名称、位置、仓位数量、状态)            │
│                                                                     │
│  Step 3: 点击目标料架所在行的 "灯测试" 按钮                         │
│          (⚡ ThunderboltOutlined 图标)                              │
│                                                                     │
│  Step 4: 前端调用 API:                                              │
│          POST /api/shelves/{shelf_id}/rack-test                     │
│          Body: { "test_mode": 15, "interval": 1000 }               │
│                                                                     │
│  Step 5: 后端处理 (shelves.py → rack_api_client.py):                │
│          a. 从 DB 加载料架信息 (404 不存在则报错)                   │
│          b. 从系统设置表读取控灯 API 配置:                          │
│             - rack_api_base_url   (控灯服务地址)                    │
│             - rack_api_user_id    (用户 ID)                         │
│             - rack_api_client_id  (客户端 ID)                       │
│          c. 创建 RackApiClient 实例                                 │
│          d. 调用 rack_test(rack_id=shelf.code, test_mode=15)       │
│                                                                     │
│  Step 6: RackApiClient 向硬件发送 HTTP 请求:                        │
│          POST {rack_api_base_url}/api/RackMgr/RackTest              │
│          Headers: userId, clientId, sessionId (UUID)                │
│          Body: { "id": shelf.code, "testMode": 15,                  │
│                  "interval": 1000 }                                 │
│          ⚠ 重试机制: 最多 3 次，指数退避 (1s, 2s, 4s)              │
│                                                                     │
│  Step 7: 智能料架硬件接收指令，开始执行灯测试序列                   │
│          → 所有槽位依次执行 RGB 循环 → 灯序 → 警示灯 → 传感器     │
│                                                                     │
│  Step 8: 硬件响应: { "code": 0, "message": "OK" }                 │
│                                                                     │
│  Step 9: 后端返回前端:                                              │
│          { "status": "ok", "shelf_id": N, "rackNo": "SMT01",       │
│            "test_mode": 15, "result": {...},                        │
│            "message": "灯测试指令已发送" }                          │
│                                                                     │
│  Step 10: 前端弹出成功提示: "灯测试指令已发送"                      │
│                                                                     │
│  Step 11: 操作员**目视检查**物理料架，确认所有 LED 正常工作        │
└─────────────────────────────────────────────────────────────────────┘
