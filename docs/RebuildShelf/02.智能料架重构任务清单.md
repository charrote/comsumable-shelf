## 一、重构背景与目标

### 1.1 背景

系统原智能料架采用 **AMKN8702G 主控板 + RS485 灯板（AMKN7141）** 方案，通过 Modbus TCP 直接控制 LED 灯并轮询储位状态。现更换为 **新智能料架**，底层硬件通信协议改为 **HTTP REST API（JSON/UTF-8）** 控灯 + 回调接口上报储位变化。

### 1.2 目标

- **保留核心业务逻辑不变**：原材料入库 → BOM发料 → 智能亮灯/FIFO出库 → 退库点计 → 日清账务
- **替换底层硬件通信链路**：Modbus TCP/RS485 → HTTP JSON WebAPI
- **平滑过渡**：保留旧系统兼容性标记，支持分步切换
- **降低风险**：新硬件 API 容错兜底，控灯离线时业务可继续

## 二、新旧架构对比

### 2.1 通信架构对比

```
旧方案（Modbus）:
┌──────────────┐   Modbus TCP (502)    ┌──────────────────┐
│  FastAPI     │ ◄────────────────────► │  AMKN8702G 主控板 │
│  Backend     │    站号: 200            │  GD32F307VE      │
│              │                         └──────┬───────────┘
│  ModbusTCPClient                      RS485 (38400bps)
│  SlotReader                        ┌───────┴───────┐
│  LEDController                     │  AMKN7141 灯板 ×N  │
│              │                      │  20/10/6 储位/板  │
│              │                      │  红/绿/蓝 LED    │
│              │                      └─────────────────┘

新方案（HTTP API）:
┌──────────────┐   HTTP POST / JSON    ┌───────────────────────────┐
│  FastAPI     │ ◄────────────────────► │  新智能料架控灯服务       │
│  Backend     │    localhost:port     │  RackCellMgr WebAPI       │
│              │                         ┌──┬──┬──┬──┐             │
│  RackApiClient│                        │灯 │警 │电 │感 │             │
│  RackSlotPoller│                       │储 │示 │量 │应 │             │
│              │                         │位 │灯 │   │传 │             │
│              │                         │ LED│   │   │测 │             │
│              │                         └──┴──┴──┴──┘             │
└──────────────┘                    ┌───┴──────────────┐           │
                                    │  感应器(空→有/有→空)│           │
                                    │  上报储位变化      │──────────► 回调
                                    └──────────────────┘
```

### 2.2 通信协议对比

| 维度 | 旧方案（Modbus） | 新方案（HTTP API） |
|------|-----------------|-------------------|
| **通信方式** | Modbus TCP 502 端口 | HTTP REST API（JSON） |
| **LED 控制** | FC15 写多线圈（地址 10000+），每 LED 4 bit | POST /LightUpCellLed（单个）/LightUpCellLedList（批量） |
| **储位读取** | FC2 读离散输入（A面:1000+, B面:2000+），1s 轮询 | 回调接口 CellChangeCallback / 查询接口 GetCellList |
| **警示灯** | 未涉及 | POST /SetWarningLed（8 种色值组合） |
| **硬件测试** | 无 | POST /TestLed（RGB/灯序/警示灯/传感器） |
| **储位查询** | 无专用 API | POST /GetCellList（分页查询） |
| **鉴权参数** | Modbus TCP IP 认证 | userId + clientId + sessionId |
| **响应格式** | 原始 Modbus 响应 | `{"code": 0, "message": "...", "sessionId": "..."}` |

## 三、智能料架关联的业务功能清单

| # | 业务模块 | 涉及功能 | 旧实现方式 | 新实现方式 |
|---|---------|---------|-----------|-----------|
| 1 | **上架（入库）** | 扫码入库 → 自动/手动分配储位 → 绑定 shelf_slot_id | SlotReader 轮询 Modbus FC2，检测到储位变化后自动绑定 | 注册回调接口接收储位变化通知；保留手动分配接口 |
| 2 | **发料出库（LED 亮灯）** | FIFO 计算 → 亮灯方案生成 → 下发 LED 指令 → 扫码出库 → 清灯 | LEDController 通过 Modbus FC15 写多线圈（地址 10000+） | RackApiClient 调用 POST /LightUpCellLedList 批量亮灯 |
| 3 | **储位状态监控** | 实时轮询所有料架储位占用/空状态 | SlotReader 每 1s 轮询 FC2 → diff 检测 → 回调业务层 | 调用 GetCellList API 定时查询；或依赖回调实时更新 |
| 4 | **退库（XR 流程）** | XR 点数 → 自动配对 → 打印标签 → 确认上架 | pairing 逻辑 + ZPL 打印机 + ShelfSlot 更新 | 同上 + 新储位绑定（cellId 格式） |
| 5 | **料架管理（配置）** | 料架 CRUD、储位管理、灯板配置 | Shelf + ShelfSlot 表，modbus_tcp_id/board_address/slot_on_board | Shelf + ShelfSlot 表简化，新增 rack_api_url/cell_prefix |
| 6 | **硬件测试与维护** | LED 测试、传感器校准 | 无 | 调用 RackTest API（testMode: RGB/灯序/警示灯/传感器）+ 电量预警 |
| 7 | **实时状态推送** | PDA/Web 实时显示储位变化、亮灯状态 | SlotReader 变化 → WebSocket 推送 | 回调接口接收变化 → 处理后 → WebSocket 推送 |

### T2 - 新增料架控灯服务封装层（RackApiClient）

**优先级**: P0  
**新建文件**: `backend/app/services/rack_api_client.py`

#### 接口封装

| 方法 | 对应 API | 参数 | 说明 |
|------|---------|------|------|
| `light_up_cell(cell_id, led_color, is_blink, turn_on_time)` | POST /LightUpCellLed | 储位号、灯色、闪烁、亮灯时间 | 单个储位亮灯 |
| `light_up_cells_batch(cells, turn_on_time, voice_text)` | POST /LightUpCellLedList | cells=列表、亮灯时间、语音 | 批量亮灯 + 语音播报 |
| `set_indicator_status(rack_id, indicator_id, indicator_status, is_blink)` | POST /SetWarningLed | 料架号、警示灯号、状态、闪烁 | 设置警示灯 |

| `cell_change_callback(data)` | POST /CellChangeCallback | 回调数据 | 响应回调（或作为中转） |

#### 实现要点

```python
class RackApiClient:
    """料架控灯 HTTP API 客户端"""
    
    def __init__(self, base_url, user_id, client_id):
        self.base_url = base_url
        self.user_id = user_id
        self.client_id = client_id
        self.session = requests.Session()
        self.timeout = 5.0  # 秒
    
    def _get_headers(self):
        return {
            "userId": self.user_id,
            "clientId": self.client_id,
            "sessionId": str(uuid.uuid4()),
            "Content-Type": "application/json;charset=utf-8"
        }
    
    def _post(self, path, data):
        """通用 POST 请求"""
        headers = self._get_headers()
        url = f"{self.base_url}{path}"
        resp = self.session.post(url, json=data, headers=headers, timeout=self.timeout)
        result = resp.json()
        if result.get("code") != 0:
            raise RackApiError(f"API 返回 code={result.get('code')}: {result.get('message')}")
        return result
    
    # 色值映射常量
    LED_COLORS = {
        "off": 0, "red": 1, "green": 2, "yellow": 3,
        "blue": 4, "magenta": 5, "cyan": 6, "white": 7
    }
    
    INDICATOR_COLORS = {
        "off": 0, "red": 1, "yellow": 2, "red_yellow": 3,
        "green": 4, "red_green": 5, "yellow_green": 6, "all": 7
    }
```

#### 容错机制

- API 不可达：记录 ERROR 日志，标记料架"控灯离线"，业务操作继续（仅不亮灯）
- 超时重试：3 次指数退避（1s, 2s, 4s）
- 异常上报：异常时通过 WebSocket 推送告警

### T5 - 改造储位状态查询（替代 SlotReader）

**优先级**: P0  
**影响文件**: `backend/app/services/slot_reader.py`、`backend/app/services/rack_api_client.py`

#### 方案 A（推荐）：轮询 + 回调双模式

**新建服务**: `backend/app/services/rack_slot_poller.py`

```python
class RackSlotPoller:
    """替代旧 SlotReader 的轮询服务"""
    
    POLL_INTERVAL = 10  # 秒（HTTP 查询，无需 1s 高频）
    
    async def start(self):
        while self._running:
            try:
                shelves = await self.db.get_active_shelves()
                for shelf in shelves:
                    client = RackApiClient(shelf.rack_api_url, ...)
                    # 分页查询全部储位
                    cells = await self._fetch_all_cells(client, shelf.id)
                    await self._sync_to_db(shelf.id, cells)
            except Exception as e:
                logger.error(f"RackSlotPoller error: {e}")
            await asyncio.sleep(self.POLL_INTERVAL)
```

- 每 10s 调用 `GetCellList` 全量查询，更新 shelf_slots 表
- 与回调结合：回调处理即时变化，轮询用于一致性校验
- 异常时降级：轮询失败不影响业务操作

#### 方案 B：纯回调模式

- 仅依赖回调更新 shelf_slots 状态
- 服务启动时一次性调用 GetCellList 同步状态
- 优势：实时性更好；风险：断连后状态不同步

#### 旧 SlotReader 处理

- 标记 deprecated，保留心跳/健康检查能力
- 删除 `read_slot_status_batch` 方法中的 Modbus 调用
- 删除 FC2 离散输入轮询逻辑

### T6 - 改造上架流程（感应自动分配）

**优先级**: P1  
**影响文件**: `backend/app/services/receipt_service.py`

#### 流程变更

```
旧流程:
```

## 八、关键设计决策

### 8.1 cell_id 格式

**格式**: `料架号(可变长) + 储位序号(4位)`

- 示例：`SHA010001`（料架 SH-A-01 储位 0001）、`A0010001`（料架 A001 储位 0001）
- 唯一约束：在 shelf_slots 表上唯一
- 迁移生成规则：shelf.code + shelf_slots.slot_on_board（补零到 4 位）

### 8.2 回调 vs 轮询策略

**推荐方案：双模式**

- **回调**：实时处理储位变化（即时性强）
- **轮询**：每 10s 同步一次（容错/一致性校验）
- 回调不可达时降级为纯轮询

### 8.3 led_color 映射

| 字符串 | integer | 用途 |
|--------|---------|------|
| "off" | 0 | 清灯 |
| "red" | 1 | 异常/拒收 |
| "green" | 2 | 正常取料引导 |
| "yellow" | 3 | 待取/注意 |
| "blue" | 4 | 特殊引导 |
| "magenta" | 5 | 特殊引导 |
| "cyan" | 6 | 特殊引导 |
| "white" | 7 | 特殊引导 |

内部代码使用字符串色值（与旧版保持一致），RackApiClient 内部转换为 integer。

### 8.4 Modbus 兼容层

- **ModbusTCPClient 保留心跳/健康检查能力**
- **不再用于 LED 控制和储位读取**
- 旧料架（未迁移）通过 old_modbus_ip 兼容访问
- 新料架通过 rack_api_url 访问

### 8.5 容错兜底策略

- **控灯 API 不可达**: 标记料架"控灯离线"，业务操作继续（仅不亮灯），WebSocket 推送告警
- **回调接口不可达**: 降级为纯轮询模式
- **轮询服务异常**: 记录日志，不影响业务操作
- **数据迁移失败**: 自动回滚到备份表
