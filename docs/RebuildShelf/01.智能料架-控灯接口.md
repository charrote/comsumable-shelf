## 1. 接口约定

| 项目 | 说明 |
|------|------|
| 协议 | HTTP（HTTPS 如有需调整） |
| 编码 | UTF-8 |
| 请求格式 | `application/json` |
| 调用模式 | 同步调用 |
| 超时 | 数据量大时需延长请求超时时间 |

> **部署说明**：料架控制服务可安装于现有服务器，建议使用虚拟机隔离安装。系统集成时仅需调用控制 WebAPI 接口。

## 2. 公共参数

### 2.1 请求公共参数

| 参数名 | 类型 | 必须 | 说明 |
|--------|------|------|------|
| `userId` | string | 是 | 操作用户 ID |
| `clientId` | string | 是 | 操作终端 ID |
| `sessionId` | string | 是 | 会话 ID（建议使用 GUID/UUID 或时间戳） |

| 参数名 | 类型 | 说明 |

## 3. 接口清单

### 3.1 点亮指定储位灯

> 使用指定颜色点亮指定储位灯

| 项目 | 值 |
|------|-----|
| **Path** | `/api/RackCellMgr/LightUpCellLed` |
| **Method** | `POST` |

#### 请求参数

| 参数名 | 类型 | 必须 | 说明 |
|--------|------|------|------|
| `cellId` | string | 是 | 储位号。格式：`料架号 + 储位序号`，如 `A0010001`。若只传料架号（如 `A001`），则点亮整个料架 |
| `ledColor` | integer | 是 | 灯色（见下方色值映射） |
| `isBlink` | boolean | 是 | 是否闪烁 |
| `turnOnTime` | integer | 否 | 延后关灯时间（秒），默认 `0` 表示不自动关灯 |
| `userId` | string | 是 | 操作人 |
| `clientId` | string | 是 | 终端设备 ID |
| `sessionId` | string | 是 | 会话 ID |

#### 请求示例

```json
{
  "cellId": "A0010001",
  "ledColor": 1,
  "isBlink": true,
  "turnOnTime": 0,
  "userId": "test",
  "clientId": "test01",
  "sessionId": "4bF67d19-EEF4-BAef-56CB-FFeAc6EB5fB4"
}
```

#### 返回数据

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `code` | number | 结果代码，`0` 成功 |
| `message` | string | 结果描述 |
| `sessionId` | string | 会话 ID |
| `turnOnTime` | integer | 延后关灯时间（秒），返回确认值 |

#### 返回示例

```json
{
  "code": 0,
  "message": "OK",
  "sessionId": "6F161C03-D1D0-4304-BA8F-FFCA03237F5C"
}
```

### 3.2 批量点亮指定储位灯

> 使用指定颜色批量点亮多个指定储位灯，支持语音播报

| 项目 | 值 |
|------|-----|
| **Path** | `/api/RackCellMgr/LightUpCellLedList` |
| **Method** | `POST` |

#### 请求参数

| 参数名 | 类型 | 必须 | 说明 |
|--------|------|------|------|
| `cells` | array[] | 是 | 储位列表，每个元素包含 `cellId`、`ledColor`、`isBlink` |
| `cells[].cellId` | string | 是 | 储位号 |
| `cells[].ledColor` | integer | 是 | 灯色 |
| `cells[].isBlink` | boolean | 是 | 是否闪烁 |
| `turnOnTime` | integer | 否 | 延后关灯时间（秒），默认 `0` |
| `voiceText` | string | 否 | 语音文本内容，如 `"非法上架"` |
| `userId` | string | 是 | 操作人 |
| `clientId` | string | 是 | 操作终端 ID |
| `sessionId` | string | 是 | 会话 ID |

#### 请求示例

```json
{
  "cells": [
    { "cellId": "A0010001", "ledColor": 1, "isBlink": true },
    { "cellId": "A0050002", "ledColor": 2, "isBlink": false }
  ],
  "turnOnTime": 0,
  "voiceText": "非法上架",
  "userId": "test",
  "clientId": "test",
  "sessionId": "6F161C03-D1D0-4304-BA8F-FFCA03237F5C"
}
```

#### 返回数据

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `code` | number | 结果代码，`0` 成功 |
| `message` | string | 结果描述 |
| `sessionId` | string | 会话 ID |

#### 返回示例

```json
{
  "code": 0,
  "message": "OK",
  "sessionId": "4bF67d19-EEF4-BAef-56CB-FFeAc6EB5fB4"
}
```

| `userId` | string | 是 | 操作人 |
| `clientId` | string | 是 | 终端设备 ID |

```json
{
  "rackId": "A001",

  "indicatorStatus": 1,
  "userId": "TEST",
  "clientId": "TEST001",
  "sessionId": "4bF67d19-EEF4-BAef-56CB-FFeAc6EB5fB4"
}
```

#### 返回数据

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `code` | number | 结果代码，`0` 成功 |
| `message` | string | 结果描述 |
| `sessionId` | string | 会话 ID |

#### 返回示例

### 3.5 储位变化回调接口（感应式料架）

> ⚠️ **此接口由接入方（业务系统）提供**，料架控制服务在用户放入/取出物料后，通过 HTTP POST 回调此接口。

| 项目 | 值 |
|------|-----|
| **Path** | 由接入方自行定义（如 `/api/CellChangedEvent`） |
| **Method** | `POST` |

#### 回调传入参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `Code` | integer | 结果代码（料架侧使用，接入方返回时用 `code`） |
| `Message` | string | 消息描述 |
| `SessionId` | string | 会话 ID |
| `ResultData` | array[] | 储位变化列表 |
| `ResultData[].CellId` | string | 储位号 |
| `ResultData[].Status` | integer | 状态：`0`-取出，`1`-放入 |
| `ResultData[].MaterialId` | string | 物料 ID（保留字段） |
| `ResultData[].SheetId` | string | 出库/入库关联单号 |
| `ResultData[].Timestamp` | datetime | 时间戳 |

#### 回调请求示例

**物料放入：**

```json
{
  "ResultData": [
    {
      "CellId": "A0010075",
      "Status": 1,
      "Timestamp": "2021-04-15T09:59:00.6825053+08:00"
    }
  ],
  "Code": 0,
  "Message": "OK",
  "SessionId": "c7021573-8c40-4c38-a44e-6d8a43935d77"
}
```

**物料取出：**

```json
{
  "ResultData": [
    {
      "CellId": "A0010075",
      "Status": 0,
      "Timestamp": "2021-04-15T09:59:01.1825053+08:00"
    }
  ],
  "Code": 0,
  "Message": "OK",
  "SessionId": "c7021573-8c40-4c38-a44e-6d8a43935d17"
}
```

#### 回调响应要求

接入方回调接口**必须**返回以下数据结构：

| 参数名 | 类型 | 必须 | 说明 |
|--------|------|------|------|
| `code` | integer | 是 | 结果代码，返回 `0` 表示接收成功 |
| `message` | string | 是 | 对 `code` 值的描述 |
| `sessionId` | string | 是 | 原请求 `sessionId`，用于流程跟踪 |

#### 回调响应示例

```json
{
  "code": 0,
  "message": "OK",
  "sessionId": "F6B6DF39-A7B4-4D4A-AAD1-1AFC0D33B799"
}
```

### 3.6 料架储位状态查询接口

> 查询料架储位信息。返回数据中包含电子标签电量信息，`< 2.5V` 为预警状态，建议更换电池。

| 项目 | 值 |
|------|-----|
| **Path** | `/api/RackCellMgr/GetRackCell` |
| **Method** | `GET`（通过 QueryString 传参） |

> **注意**：文档中同时展示了 JSON Body 传参示例，但 Method 标注为 GET。建议兼容两种方式，优先使用 QueryString。

#### 请求参数

| 参数名 | 类型 | 必须 | 说明 |
|--------|------|------|------|
| `RackId` | string | 否 | 料架号 |
| `Filter` | string | 否 | 储位号筛选 |
| `PageIndex` | integer | 是 | 页码，默认 `1` |
| `PageSize` | integer | 是 | 每页记录数，默认 `10` |

#### 请求示例

```
GET /api/RackCellMgr/GetRackCell?RackId=A004&PageIndex=1&PageSize=5
```

#### 返回数据

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `resultData` | object | 分页数据 |
| `resultData.total` | number | 总记录数 |
| `resultData.items` | array[] | 储位列表 |
| `resultData.items[].id` | number | 记录 ID |
| `resultData.items[].rackId` | string | 料架号 |
| `resultData.items[].cellId` | string | 储位号 |
| `resultData.items[].ledId` | number | LED ID |
| `resultData.items[].rackType` | number | 料架类型 |
| `resultData.items[].ledColor` | number | 当前灯色 |
| `resultData.items[].blink` | number | 闪烁状态（0/1） |
| `resultData.items[].side` | number | 侧面 |
| `resultData.items[].needChk` | number | 是否需盘点 |
| `resultData.items[].status` | number | 状态 |
| `resultData.items[].used` | number | 是否使用：`0`-空，`1`-非空 |
| `resultData.items[].createTime` | string | 创建时间 |
| `resultData.items[].updateUser` | string | 最后更新人 |
| `resultData.items[].description` | string/null | 描述 |
| `resultData.items[].item` | string | 巷道灯 ID |
| `resultData.items[].bluetooth_id` | string | 电子标签 ID |
| `resultData.items[].electricitys` | string/null | **电量**，`< 2.5V` 为预警点 |
| `code` | number | 结果代码，`0` 成功 |
| `message` | string | 结果描述 |
| `sessionId` | string | 会话 ID |

#### 返回示例

### 4.3 硬件测试模式（RackTest API）

> 对应 3.4 节 `POST /api/RackMgr/RackTest` 接口，通过 HTTP API 下发测试指令，不依赖 Modbus TCP 直连。

供 3.4 节接口使用：

|| testMode | 模式 | 说明 |
||----------|------|------|
|| `0` | 取消测试 | 退出测试模式，停止所有正在进行的测试 |
|| `1` | RGB 灯珠测试 | 测试所有 LED 灯珠颜色（逐色循环） |
|| `2` | 灯序测试 | 测试灯序（依次亮灭） |
|| `4` | 警示灯测试 | 测试警示灯（8 种色值组合循环） |
|| `8` | 感应式传感器测试 | 测试红外传感器（空→有/有→空 状态切换） |
|| *(可组合)* | 全部测试 | 如 `1+2+4+8=15` 全部测试 |

> **参数说明**：接口同时支持 `interval` 参数控制测试变化间隔（秒），传 `0` 使用默认值。
> **注意**：测试通过 HTTP API 远程下发，不再需要 Modbus TCP 连接主控板。

## 5. 接口调用流程参考

```
┌─────────────┐         ┌──────────────────┐         ┌──────────────┐
│   业务系统   │ ──────> │  料架控制服务     │ <────── │ 感应式料架硬件 │
│ (Comsumable) │  HTTP   │  (WebAPI 端口)    │   POST   │  (储位感应)    │
└─────────────┘         └──────────────────┘         └──────────────┘
      │                           │
      │ POST /LightUpCellLed      │ 回调 /CellChangedEvent
      │ POST /LightUpCellLedList  │ (业务系统需实现此端点)
      │ POST /SetIndicatorStatus
      │ POST /RackTest
      │ GET  /GetRackCell
```

## 6. 实施注意事项

1. **储位号格式**：`料架号(4位) + 储位序号(4位)`，如 `A0010001` = 料架 A001 第 0001 号储位
2. **警示灯自动同步**：默认储位灯颜色会自动同步到警示灯，无需手动调用 SetIndicatorStatus
3. **电池预警**：查询储位状态时，`electricitys < 2.5V` 需提醒更换电池
4. **超时处理**：批量操作时需延长 HTTP 超时时间
5. **Session 追踪**：所有请求/响应均携带 `sessionId`，用于请求链路追踪
