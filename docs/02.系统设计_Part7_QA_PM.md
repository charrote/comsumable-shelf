## 1. 测试策略总览

```
单元测试 (Unit Test)
  └── 业务逻辑: FIFO 算法、防呆校验、XR 配对
  └── 工具函数: 条码解析、LED 字节计算
  └── 覆盖率目标: > 80%

  ├── API 端到端: POST /receipts/scan → DB 写入验证

  └── XR 模拟: 模拟 XR 数据上传

UI 测试 (E2E)
  └── PDA 扫码流程: 入库 → 发料 → 退库 → 日报
  
性能测试
  ├── API 响应: < 500ms (95th percentile)
  ├── 亮灯延迟: < 1s (Modbus 指令到达)
  └── 并发 PDA: 支持 10 人同时操作
```

| FIFO-02 | 尾数优先 - 多盘凑齐 | 需求 100, 库存 [30,50,40] | 取盘#1(30)+#2(50)+#3(20) | P0 |

| FIFO-05 | 时间 FIFO - 早先出 | 需求 100, 库存 [40@早,50@中,30@晚,100@最新] | 取#1(40)+#2(50)+#3(10) | P0 |
| FIFO-06 | 混合策略 - 同量按时间 | 需求 50, 库存 [30@早,30@晚,80@最早] | 取#1(30)+#2(20) | P1 |

| FIFO-10 | Config 策略切换 | 系统配置改为 time_fifo | 按 last_in_time 排序出库 | P1 |

## 3. API 测试用例

### 3.1 入库 API

```python
def test_receipt_scan_success():
    """正常扫码入库"""
    resp = post("/api/v1/receipts/scan", {
        "barcode": "ABC-001-A20260616-001",
        "operator": "operator01"
    })
    assert resp.status == 200
    assert resp.data["action"] == "first_in"
    assert resp.data["duplicate_flag"] is False
    assert resp.data["inventory_pallet_id"] is not None

def test_receipt_scan_duplicate():
    """重复扫码入库"""
    post("/api/v1/receipts/scan", {
        "barcode": "ABC-001-A20260616-001",
        "operator": "operator01"
    })
    resp = post("/api/v1/receipts/scan", {
        "barcode": "ABC-001-A20260616-001",
        "operator": "operator01"
    })
    assert resp.status == 200
    assert resp.data["status"] == "duplicate"
    assert resp.data["matched_pallet_id"] is not None

def test_receipt_scan_invalid_barcode():
    """无效条码"""
    resp = post("/api/v1/receipts/scan", {
        "barcode": "INVALID",
        "operator": "operator01"
    })
    assert resp.status == 400
```

### 3.2 出库 API

```python
def test_issue_confirm_pick():

        "operator": "operator01"

    assert resp.data["status"] == "duplicate_out"

    """全部取完"""

        "barcode": "ABC-001-A20260616-001",
        "pallet_id": 5001,
        "operator": "operator01"
    })
    assert resp.data["status"] == "ok"
    assert resp.data["all_picked"] is True
    assert resp.data["cleared_leds"] is not None
```

### 3.3 报表 API

```python
def test_daily_report_calculation():
    """日报计算验证"""
    resp = get("/api/v1/reports/daily?date=2026-06-16")
    assert resp.data["report_date"] == "2026-06-16"
    assert resp.data["summary"]["total_in"] == 250
    assert resp.data["summary"]["total_out"] == 180
    assert resp.data["summary"]["total_balance"] == 70
    # 验算: 结余 = 在库 - 在线
    for detail in resp.data["details"]:
        assert detail["closing_balance"] == detail["in_qty"] + \
            detail["opening_balance"] - detail["out_qty"]
```

## 5. PM 验收标准 (Acceptance Criteria)

### 5.1 功能验收清单

| 模块 | 功能点 | AC 标准 | 状态 |
|------|--------|---------|------|
| **入库** | 扫码录入 | 扫码 → 物料识别 → 入库成功 ≥ 99% | ☐ |
| **入库** | 防重码 | 重复扫码 100% 拦截 | ☐ |
| **发料** | BOM 导入 | Excel 解析准确率 ≥ 99% | ☐ |
| **发料** | BOM 替代料 | 替代料关系正确解析 | ☐ |
| **发料** | 亮灯指令 | 发料单下发 → 料架亮灯 ≤ 2s | ☐ |
| **出库** | FIFO 策略 | 尾数优先 / 时间 FIFO 正确 | ☐ |
| **出库** | 出库扫码 | 扫码 → 确认 → 库存更新 准确 | ☐ |
| **出库** | 出库防呆 | 重复出库 100% 拦截 | ☐ |
| **退库** | XR 配对 | ±5s 窗口内自动配对 ≥ 95% | ☐ |
| **退库** | 二次上架 | 点数 → 配对 → 上架 链路完整 | ☐ |
| **账务** | 实时更新 | 出入库后库存 ≤ 1s 更新 | ☐ |
| **账务** | 日报表 | in_stock/online/balance 准确 | ☐ |
| **账务** | 客户分类 | 按客户统计准确 | ☐ |
| **离线** | 断网操作 | 断网时核心操作可用 | ☐ |
| **离线** | 断网恢复 | 网络恢复后自动同步 | ☐ |

### 5.2 非功能验收

| 维度 | 标准 | 测量方式 |
|------|------|----------|
| 可用性 | 月故障时间 < 4h (99.5%) | 运维日志 |
| 数据准确性 | 账务误差率 = 0 | 盘点对账 |
| 硬件响应 | 亮灯延迟 < 1s | 示波器实测 |

## 6. UAT (用户验收测试) 脚本

### 场景 1: 完整入库→发料→出库→退库→日报 流程

```
Step 1: 创建客户 "测试客户"
Step 2: 创建物料 "ABC-001" (PCB板 A型, 标准盘量 50)
Step 3: 扫码入库 3 盘: 50盘、40盘、30盘
Step 4: 上传 BOM, 生成发料单 (需求 100 盘)
Step 5: 计算亮灯方案 (尾数优先) → 应选: 30+40+30
Step 6: 下发亮灯 → 料架灯正确亮起
Step 7: PDA 扫码出库 30 盘 → 灯灭, 库存更新
Step 8: 继续扫码出库 → 全部出库完成
Step 9: 退库 20 盘 → XR 点数 → 自动配对
Step 10: 二次上架 → 库存更新
Step 11: 查看日清表 → in_stock/online/balance 正确
```

### 场景 2: 异常流程

```
Step 1: 重复扫码入库 → 应拦截 (无主管授权选项)
Step 2: 重复扫码出库 → 应拦截 (无主管授权选项)
Step 3: 非本单物料扫码 → 应拦截
Step 4: 网络断开 → 应进入离线模式
Step 5: XR 点数超时 → 应提示手动配对
```
