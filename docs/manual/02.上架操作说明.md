# PDA 上架操作说明

> 适用端：PDA
> 对应角色：仓库操作员
> 最后更新：2026-06-27
> 关联版本：v1.0

---

## 一、业务目标

将已收料的库存盘（Reel）放入智能料架的指定储位，并在系统中记录"哪个盘在哪个架子的哪个储位上"。

上架完成后：
- 库存盘关联到具体的 **料架编号 + 储位编号**
- 可通过库存查询精确定位到物理位置
- 出库时可控制对应储位 **LED 亮灯引导** 取料

### 料架通信架构

系统通过 **HTTP REST API（JSON/UTF-8）** 与智能料架控制服务通信：

| 通信方式 | 说明 |
|---------|------|
| **控灯接口** | 后端通过 `RackApiClient` 调用料架 HTTP API 控制 LED 灯（`POST /api/RackCellMgr/LightUpCellLed` 等） |
| **储位变化回调** | 料架传感器检测到储位变化时，主动向后端 `POST /api/rack/callback/cell-changed` 推送事件 |
| **轮询兜底** | `RackSlotPoller` 后台服务每 10 秒调用 `GetCellList` 全量查询储位状态，确保数据一致 |

每个储位对应一个唯一的 **Cell ID**（如 `A001A0001`），作为料架与系统之间的统一标识。

## 二、PDA 操作入口

| 操作 | 说明 |
|------|------|
| PDA 底部导航 → **料盘上架** | 进入上架主界面 |

PDA 支持两种上架模式，通过顶部 Tab 切换：

| 模式 | 适用场景 |
|------|---------|
| **智能料架**（推荐） | 配备储位传感器的智能料架，放入即自动识别 |
| **手动上架** | 无传感器料架或传感器故障时的备选方案 |

## 三、前置条件

| 条件 | 说明 |
|------|------|
| 收料已完成 | 库存盘已创建，且状态为 `on_shelf` 或 `pending_shelving` |
| 空储位可用 | 目标料架有未占用的储位 |
| 料架通信正常 | 智能料架模式下，料架控灯服务网络可达（HTTP API） |

## 四、智能料架上架（推荐）

### 操作流程

智能料架模式基于 **回调驱动 + WebSocket 推送** 设计，操作员只需要两步：

```
扫描料盘条码  →  放入任意空储位（传感器自动检测 + 后端自动绑定）
     ①                        ②
```

**不需要选择料架，不需要点击保存**——放入后系统自动完成绑定，PDA 实时接收到结果。

---

### Step 1：扫描料盘

| PDA 操作 |
|----------|
| 进入"料盘上架"界面，确保顶部 Tab 选中「智能料架」 |
| 扫描收料时打印的内部标签条码（或手动输入条码后回车） |
| 系统调用 `POST /shelving/scan` 查询该 Reel 信息 |

**后台行为**：
1. 系统查到该 Reel 且未绑定储位
2. 自动将 Reel 状态标记为 **`pending_shelving`**（待上架），等待回调自动绑定
3. PDA 自动连接后端 **WebSocket**（`ws://host:port/ws`），监听 `reel_bound` 事件

**PDA 显示**（面板 I - 扫描料盘）：
```
┌──────────────────────────────────┐
│  I  扫描料盘                      │
│                                   │
│  Reel #1024          [待上架]     │
│  物料编码: RES-0402-1001         │
│  物料名称: 贴片电阻 100Ω          │
│  数量: 1                          │
└──────────────────────────────────┘
```

> **状态说明**：扫描后若该盘已上架（`already_bound`），系统会直接提示，无需重复操作。

---

### Step 2：放入空储位

操作员将 Reel 放入**任意料架的任意空储位**。不需要指定料架，不需要扫码储位。

**PDA 显示**（面板 II - 等待放入）：
```
┌──────────────────────────────────┐
│  II  储位识别          [回调]   │
│                                   │
│   等待放入料架...                 │
│   请将料盘放入任意料架的空储位    │
│   传感器检测到放入后将自动绑定    │
│                                   │
│   ● 已连接                        │
└──────────────────────────────────┘
```

### 系统自动执行流程

放入储位后，以下流程**全自动完成**，无需 PDA 操作：

```
操作员放入料盘
       │
       ▼
┌─────────────────────────────────────────┐
│ ① 料架传感器检测到储位从「空→占用」      │
│    主动向后端发起 HTTP 回调               │
│    POST /api/rack/callback/cell-changed  │
│    {"data":[{"cellId":"SHA01A0005",      │
│              "status":1,                 │
│              "timestamp":"..."}]}        │
└────────────────┬────────────────────────┘
                 ▼
┌─────────────────────────────────────────┐
│ ② 后端 rack_callback 处理               │
│    - 查找 cell_id → ShelfSlot           │
│    - 调用 _find_unbound_reel()           │
│      找最旧的 pending_shelving 料盘      │
│    - 自动绑定:                           │
│      reel.shelf_slot_id = slot.id       │
│      reel.status = "on_shelf"           │
│    - 记录 ShelfSlotEvent                │
│    - WebSocket 广播 reel_bound 事件      │
└────────────────┬────────────────────────┘
                 ▼
┌─────────────────────────────────────────┐
│ ③ PDA 通过 WebSocket 收到 reel_bound    │
│    - 匹配 reel_id                       │
│    - 显示上架成功界面                    │
│    - 展示料架编码 + 储位编码             │
└─────────────────────────────────────────┘
```

**PDA 显示**（绑定成功后自动跳转）：
```
┌──────────────────────────────────┐
│          ✅ 上架完成              │
│                                   │
│   上架成功!                       │
│   料架: SH-A-01                  │
│   储位: A0005                    │
│                                   │
│       [继续上架]                  │
└──────────────────────────────────┘
```

> **整个智能料架模式，操作员只需要两步：扫码 → 放入。** 不需要选料架、不需要点保存。

---

### 超时兜底

如果 60 秒内未收到回调通知（传感器未触发或通信异常），PDA 会显示超时提示：

```
┌──────────────────────────────────┐
│  II  储位识别                     │
│                                   │
│   ⏱ 检测超时                     │
│   传感器未检测到放入              │
│   请确认料架通信正常              │
│                                   │
│   [重试]  [切换手动模式]          │
└──────────────────────────────────┘
```

操作员可选择：
- **重试** — 重新进入等待状态
- **切换手动模式** — 转为扫码储位条码完成上架

### RackSlotPoller 后台兜底

`RackSlotPoller` 后台服务每 10 秒调用 `GetCellList` 全量同步所有储位状态到数据库，确保回调漏报时数据仍能一致。但 PDA 不依赖此轮询——储位变化通过 WebSocket 实时推送。

## 五、手动上架

### 适用场景

- 料架未配备储位传感器
- 传感器故障或通信中断
- 传感器模式检测失败时的备选方案

### 操作流程

```
扫描料盘条码  →  扫描储位条码  →  确认上架
     ①                  ②               ③
```

---

### Step 1：扫描料盘

| PDA 操作 |
|----------|
| 顶部 Tab 切换到「手动上架」 |
| 扫描 Reel 上的标签条码 |

操作与智能料架 Step 1 相同，扫描后显示料盘信息。

> 手动模式下不会启动传感器轮询，完全依赖扫码操作。

---

### Step 2：扫描储位条码

| PDA 操作 |
|----------|
| 使用 PDA 摄像头扫描目标储位上粘贴的**储位条码标签** |
| 条码格式如 `A1A05`（料架编码 + 面 + 位号），系统自动解析出对应的料架和储位 |

**支持的条码格式**（大小写均可）：

| 格式 | 示例 | 说明 |
|------|------|------|
| 紧凑格式 | `A1A05` | 料架码 + 面 + 储位号（推荐） |
| 储位号前加横杠 | `A1-A05` | 面与储位号间加横杠 |
| 面和位间加横杠 | `A1A-05` | 面与储位号间加横杠 |
| 全部分隔 | `A1-A-05` | 每段用横杠分隔 |

**储位条码解析规则**：
```
A1  A  05
 ├──┬──┬──
 料架 面  储位号
 编码     2位数字编号
```

**系统校验流程**：
1. 调用 `POST /shelving/scan-slot` 解析条码 → 查找对应的料架和储位
2. 校验该储位是否存在
3. 校验该储位是否已被占用

**PDA 显示**（扫描储位成功后，面板 II）：
```
┌──────────────────────────────────┐
│  II  储位识别          [扫码]   │
│                                   │
│  📦                               │
│  料架: SH-A-01                   │
│  储位: A05                       │
└──────────────────────────────────┘
```

> 如果储位已被占用，系统会提示"储位被占用"，需选择其他空储位。

---

### Step 3：确认上架

确认料盘信息和储位信息无误后，点击「保存」。

系统执行绑定逻辑（调用 `POST /shelving/bind`），成功后显示上架完成界面。

## 六、扫描界面说明

PDA 使用内置摄像头进行条码扫描：

| 操作方式 | 说明 |
|---------|------|
| 📷 摄像头图标 | 点击打开摄像头扫码界面，对准条码自动识别 |
| 输入框 + 回车 | 手动输入条码后按回车提交，支持外接蓝牙扫描枪 |

- **料盘扫描**：扫描收料时打印的内部标签（或料盘上已有的可识别条码）
- **储位扫描（手动模式）**：扫描料架储位旁粘贴的储位条码标签

## 七、操作流程对比

| 步骤 | 智能料架模式 | 手动上架模式 |
|------|------------|------------|
| ① 扫描料盘 | ✅ 必须 | ✅ 必须 |
| ② 识别储位 | 传感器自动检测（放入即检测，WebSocket 推送） | 扫描储位条码标签 |
| ③ 确认上架 | ❌ 不需要（回调自动绑定） | ✅ 点击「保存」 |
| 依赖传感器 | ✅ 需要料架传感器 + HTTP 回调 | ❌ 不需要 |
| Cell ID 格式 | 自动解析（如 `SHA01A0005`） | 通过储位条码映射 |

## 八、数据流参考

### 涉及 API

| API | 方法 | 说明 |
|-----|------|------|
| `/shelving/scan` | POST | 扫描料盘条码，返回料盘信息；标记 Reel 为 `pending_shelving` |
| `/shelving/scan-slot` | POST | （手动模式）扫描储位条码，解析并校验储位 |
| `/shelving/bind` | POST | （手动模式）绑定料盘到指定储位 |
| `/api/rack/callback/cell-changed` | POST | （料架→系统）料架传感器检测到储位变化时主动回调 |
| `ws://host:port/ws` | WebSocket | 实时接收 `reel_bound` 事件（智能模式） |

### 料架控灯服务 API（由后端 RackApiClient 调用）

| API | 用途 | 说明 |
|-----|------|------|
| `POST /api/RackCellMgr/GetCellList` | 查询储位状态 | `RackSlotPoller` 每 10 秒调用（兜底同步） |
| `POST /api/RackCellMgr/LightUpCellLed` | 单储位亮灯 | 出库扫码清灯时调用 |
| `POST /api/RackCellMgr/LightUpCellLedList` | 批量亮灯 + 语音 | FIFO 出库分配时调用 |
| `POST /api/RackCellMgr/CellChangeCallback` | 储位变化回调 | 料架→系统的反向通知 |

### 涉及数据库表

| 表 | 操作 | 说明 |
|----|------|------|
| `inventory_reels.status` | UPDATE | 扫描时标记为 `pending_shelving`；绑定后更新为 `on_shelf` |
| `inventory_reels.shelf_slot_id` | UPDATE | 绑定 Reel 到储位 |
| `shelf_slot_events` | INSERT | 记录储位变化事件（bound/occupied/released） |
| `shelf_slots.last_sensor_state` | UPDATE | 由回调或轮询更新传感器状态 |

### 上架后的数据状态

```
InventoryReel #1024
  ├── status = "on_shelf"
  ├── shelf_slot_id = 42          ← 绑定到储位
  └── material_id → MaterialMaster

ShelfSlot (id=42)
  ├── shelf_id → Shelf (code="SH-A-01")
  ├── side = "A"
  ├── slot_on_board = 5
  └── cell_id = "SHA01A0005"      ← 料架统一储位标识（料架码+面+4位序号）
```

## 九、异常处理

| 异常 | 系统行为 | 操作员处理 |
|------|---------|-----------|
| 料盘已上架 | 扫描后提示"该料盘已上架至 XX/XX" | 无需操作，或先解绑再重新上架 |
| 储位已被占用 | 回调时检测到占用，跳过该储位等待下一个变化 | 放入其他空储位 |
| 传感器无响应 | 智能模式 60 秒超时 | 点击「切换手动模式」扫码绑定，或检查料架通信 |
| 放入后回调未触发 | 超时提示，可能传感器或网络异常 | 取出后重新放入；或切手动模式 |
| 料架控灯服务离线 | 回调不可用，超时 | 切换至手动模式扫码上架 |
| 料盘未入库 | 提示"未找到该料盘，请先入库" | 先完成收料流程 |
| WebSocket 断开 | 自动重连；重连后需重新扫描料盘 | 检查 PDA 网络连接 |
| 储位条码格式无效 | 提示"储位条码格式无效，示例: A1A05" | 检查条码标签是否清晰完整 |
| 网络断开 | PDA 请求失败，提示重试 | 检查 PDA 网络连接后重试 |

## 十、常见问题

**Q: 智能料架传感器检测不到怎么办？**
A: 确保 Reel 完全放入储位底部。如果多次尝试仍检测不到，请切换到「手动上架」模式，通过扫描储位条码完成绑定。

**Q: 智能料架模式需要扫描储位条码吗？**
A: 不需要。智能模式下所有料架的储位都通过传感器自动检测，放入后系统自动通过回调 + WebSocket 完成绑定。**不需要扫码储位，不需要点保存。**

**Q: PDA 放入了储位但没反应怎么办？**
A: PDA 通过 WebSocket 接收 `reel_bound` 事件，等待最多 60 秒。如果超时未收到，请确认：
1. 料架传感器是否正常（储位放入后 LED 灯是否变化）
2. 料架与系统的网络通信是否正常
3. 后端服务是否运行正常
如确认传感器已触发仍无反应，可切换到「手动上架」模式。

**Q: 如果同时多台 PDA 扫描料盘，放入一个储位会绑定哪个盘？**
A: 回调接口按 `created_at ASC` 顺序查找最旧的 `pending_shelving` 状态的 Reel 进行绑定。先扫描的料盘优先被绑定。

**Q: 一个储位可以放多盘料吗？**
A: 当前系统默认一个储位绑定一盘料。如果需要放置多盘，需另行配置。

**Q: 上架后想换到另一个储位怎么办？**
A: 在 PC Web 端「库存管理」中找到该盘 → 修改储位 → 选择新的空储位。原储位自动释放。

**Q: 上架成功后料架 LED 灯会亮吗？**
A: 上架操作本身不触发 LED 亮灯。LED 亮灯在**出库环节**由 FIFO 计算分配后通过 `POST /api/RackCellMgr/LightUpCellLedList` 触发。

**Q: PDA 上如何重新开始上架流程？**
A: 完成上架后点击「继续上架」，或在任意步骤点击「重新扫描」重置当前料盘信息。
