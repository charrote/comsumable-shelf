# 点料操作说明

> 适用端：PDA + XR 点料机
> 对应角色：仓库操作员
> 最后更新：2026-06-17
> 关联版本：v1.0

---

## 一、业务目标

对已出库（产线退库）的物料盘进行 **点数 → 确认数量 → 重新上架**，实现物料的循环利用。

点料流程解决了 SMT 行业的经典问题：
- 产线退回的料盘（Reel）剩余数量未知
- 需要高精度点数才能重新入库
- 点数后重新打印标签，更新数量

---

## 二、完整操作流程

```
产线退料 ──→ 出库（设置为tracking） ──→ XR点料机点数 ──→ 系统配对 ──→ 确认上架
   ①                   ②                        ③              ④          ⑤
```

### 前置条件

| 条件 | 说明 |
|------|------|
| 物料盘已出库 | 库存盘状态为 `tracking`（退库跟踪中） |
| XR 点料机已连接 | 通过蓝牙/串口/WiFi 连接 PDA |
| 标签打印机已配置 | 点数后打印新标签（可选） |

---

## 三、分步操作

### Step 1：产线退料（系统外）

产线将使用完毕的物料盘退回仓库。

> 退料时，操作员在 PDA 上执行"退库"操作，
> 将对应的库存盘状态从 `in_use` 或 `on_shelf` 改为 `tracking`。

### Step 2：出库设置为 tracking

| PDA 操作 |
|----------|
| 退库 → 扫描 Reel 条码 |
| 系统将 Reel 状态设为 `tracking`，记录出库时间和关联订单 |

### Step 3：XR 点料机点数

| 操作 |
|------|
| 操作员将退库的料盘放入 XR 点料机 |
| XR 点料机自动扫描盘上的条码（Reel ID）并计算剩余数量 |
| PDA 接收 XR 数据：`reel_id` + `counted_qty` |
| PDA 将数据上传到系统 |

**PDA 显示**：
```
┌───────────────────────────────┐
│  📡 XR 点料上传                │
│                                │
│  Reel ID: ABC-12345            │
│  识别物料: RES-0402-1001       │
│  点料数量: 850                 │
│  状态: 配对中...               │
│                                │
│  系统自动匹配退库记录...        │
└───────────────────────────────┘
```

### Step 4：系统自动配对

系统收到 XR 数据后，自动在 `tracking` 状态的库存盘中查找匹配记录。

| 匹配条件 | 说明 |
|---------|------|
| 物料编码相同 | 同一种物料 |
| 时间窗内（默认 ±5 秒） | 从出库到点料的间隔在时间窗口内 |

**配对成功**：
```
┌───────────────────────────────┐
│  ✅ 配对成功                   │
│  库存盘 #: 1024                │
│  物料: RES-0402-1001           │
│  原始数量: 1000                │
│  点料数量: 850                 │
│  差异: -150（已使用）          │
│                               │
│  [确认上架]  [重新点数]        │
└───────────────────────────────┘
```

**配对失败**：
```
┌───────────────────────────────┐
│  ⚠️ 配对失败                   │
│  未找到匹配的退库记录           │
│                               │
│  [人工配对]                    │
└───────────────────────────────┘
```

配对失败时，需操作员在系统中手动选择对应的退库记录。

### Step 5：确认上架

| 操作 |
|------|
| 确认 XR 点料数量正确 |
| 选择要放入的目标储位（或在感应料架上自动绑定） |
| 系统更新库存数量为点料结果 |
| 系统打印新的内部标签 |

**系统执行**：
1. 更新 `InventoryReel.quantity = 850`（点料结果）
2. 更新 `InventoryReel.status = on_shelf`
3. 更新 `InventoryReel.inbound_type = restock`
4. 更新 `InventoryReel.last_in_time = now()`
5. 绑定储位（如果选择了）
6. 记录 Transaction（type=restock）
7. 打印新标签

**新标签内容**：
```
┌───────────────────┐
│  内部料号: RES-01   │
│  客户料号: ABC-123  │
│  Reel: 1024        │
│  QTY: 850          │
└───────────────────┘
```

## 四、人工配对

当系统自动配对失败时，操作员可手动配对：

| PDA 操作 |
|----------|
| 1. 在"待配对"列表中选择一个 XR 批次记录 |
| 2. 选择或扫描对应的退库库存盘 |
| 3. 确认配对 |

---

## 五、数据流参考

### 5.1 涉及的表

| 表 | 操作 | 说明 |
|----|------|------|
| `inventory_pallets` | UPDATE | `quantity`<点料结果>, `status=on_shelf`, `inbound_type=restock` |
| `xr_batches` | INSERT | 记录 XR 点料机上传数据 |
| `xr_batches` | UPDATE | `matched_pallet_id`, `status=matched` |
| `transactions` | INSERT | `type=restock`, `source_type=xr_transfer` |

### 5.2 库存盘状态流转

```
          出库(料单/单独)
on_shelf ────────────────→ tracking ──→ XR点数 ──→ on_shelf (退库再上架)
  │                          │
  │                          └──→ exhausted (消耗完)
  └──→ exhausted (直接耗完)
```

## 六、异常处理

| 异常 | 系统行为 | 操作员处理 |
|------|---------|-----------|
| 自动配对失败 | 创建未匹配的 XR 批次记录 | 进入人工配对流程，手动选择库存盘 |
| 点料数量与原始数量差异大 | 系统提示差异值，不阻断流程 | 确认数量是否正常（可能是已使用部分） |
| XR 设备离线 | PDA 提示连接失败 | 检查蓝牙/WiFi 连接，重新上传 |
| 多个退库盘在同一时间窗口 | 选择匹配度最高的（时间差最小） | 检查匹配结果，不匹配则人工修正 |
| 打印机离线 | 上架成功，标签未打印 | 在记录中手动点击"重打标签" |

## 七、常见问题

**Q: 可以不经过 XR 点料直接退库上架吗？**
A: 系统支持直接退库上架（不经过 XR 点数），但数量以退库时记录的为准。建议经过 XR 精确点数以确保库存准确。

**Q: 点料后发现数量不对怎么办？**
A: 在确认上架前可选择"重新点数"。如果已确认上架，可在库存管理中手动调整数量（需 supervisor 权限）。

**Q: 退库的盘和原储位不一致怎么办？**
A: 上架时可以选择新的空储位，不一定要放回原储位。

**Q: XR 点料机能识别所有类型的条码吗？**
A: XR 设备通常支持常见的 1D/2D 条码格式。系统会对 XR 上传的 reel_id 进行物料匹配，不匹配时可人工介入。
