# 单独出库操作说明

> 适用端：PDA / PC Web
> 对应角色：仓库操作员
> 最后更新：2026-06-17
> 关联版本：v1.0

---

## 一、业务目标

**不需要 BOM 料单**，直接扫描一个库存盘（Reel）将其从库存中移出。

适用场景：
- **退料** — 产线退回多余物料，直接出库退给供应商
- **废料处理** — 过期/损坏的物料盘需要从系统移除
- **样品取料** — 研发/品质部门取少量样品
- **紧急出库** — 产线急需某一盘料来不及走 BOM 流程

---

## 二、与料单出库的区别

| 维度 | 料单出库 | 单独出库 |
|------|---------|---------|
| 前置条件 | 需 BOM + 发料单 + FIFO 计算 + LED 分配 | **无需任何前置** |
| 操作步骤 | 6 步（BOM上传→生成→计算→亮灯→取料→确认） | **2 步**（扫 Reel → 确认） |
| 出库记录 | 关联发料单号 | 仅记录 Transaction，无发料单 |
| 库存影响 | 可部分消耗 | 整盘出库（整盘 quantity 归零） |
| 适用场景 | 批量按 BOM 发料 | 零星/异常出库 |

## 三、操作流程

### 3.1 PDA 操作

| Step | 操作 | 说明 |
|------|------|------|
| 1 | 出库 → 选择"直接出库" | 进入单独出库模式 |
| 2 | 扫描 Reel 上的条码 | 扫描待出库盘的标签 |
| 3 | 确认出库信息 | 系统显示物料编码、数量、储位信息 |
| 4 | 点击"确认出库" | 系统执行出库操作 |

**PDA 显示**：
```
┌───────────────────────────────┐
│  📦 直接出库                   │
│                                │
│  扫描结果:                      │
│  库存盘 #: 1024                │
│  物料编码: RES-0402-1001       │
│  物料名称: 贴片电阻 100Ω        │
│  当前数量: 30                   │
│  储位: SH-A-01-05              │
│                                │
│  [确认出库]  [取消]             │
└───────────────────────────────┘
```

### 3.2 PC Web 操作

| Step | 操作 |
|------|------|
| 1 | 库存管理 → 找到目标 Reel |
| 2 | 点击操作列的"直接出库"按钮 |
| 3 | 在弹出的确认框中确认 |

### 3.3 确认后系统行为

| 系统动作 | 说明 |
|---------|------|
| 1. 校验 Reel 存在且 `status = on_shelf` | 已出库的盘不可再次出库 |
| 2. 库存盘 `quantity = 0`, `status = exhausted` | 整盘出库 |
| 3. 记录 Transaction（type=out） | 交易流水：单独出库 |
| 4. 释放储位（可选） | `shelf_slot_id = null`，取决于系统配置 |
| 5. 熄灭该储位 LED（如果有） | 避免残留亮灯 |
| 6. 返回成功消息 | |

## 四、数据流参考

### 4.1 涉及的表

| 表 | 操作 | 说明 |
|----|------|------|
| `inventory_pallets` | UPDATE | `quantity=0`, `status=exhausted` |
| `transactions` | INSERT | `type=out`, `source_type=direct_out` |
| `shelf_slots` | （可选） | 释放储位时更新 |

### 4.2 出库前后状态

```
出库前:
  InventoryReel #1024
    quantity = 30
    status = "on_shelf"
    shelf_slot_id = 42

出库后:
  InventoryReel #1024
    quantity = 0
    status = "exhausted"
    shelf_slot_id = null   ← 储位释放
```

## 五、异常处理

| 异常 | 系统行为 | 操作员处理 |
|------|---------|-----------|
| Reel 已出库 | 拒绝，提示"该盘已出库，状态为 exhausted" | 检查是否重复操作 |
| Reel 状态为 tracking | 拒绝，提示"该盘正在退库跟踪中，请先完成退库" | 先完成 XR 点料退库流程 |
| Reel 不存在 | 拒绝，提示"未找到该库存盘" | 检查条码是否正确 |
| 输入确认后网络断开 | PDA 本地缓存，上线后自动提交 | 确认缓存队列状态 |

## 六、常见问题

**Q: 单独出库后还能撤销吗？**
A: 不能直接撤销。但可人工创建一个新的入库单，将物料重新入库（反操作）。

**Q: 单独出库会影响 FIFO 计算吗？**
A: 不会。FIFO 计算时自动过滤 `status = exhausted` 的盘。

**Q: 单独出库需要什么权限？**
A: 建议限制为 `supervisor` 及以上角色操作，避免误操作导致库存丢失。

**Q: 出库时可以选择只出一部分数量吗？**
A: 当前版本单独出库为**整盘出库**，数量全部归零。如需部分出库请使用料单出库流程。
