# 料单出库操作说明

> 适用端：PDA（主） / PC Web
> 对应角色：仓库操作员
> 最后更新：2026-06-17
> 关联版本：v1.0

---

## 一、业务目标

根据 BOM 需求，按 **FIFO 原则**从料架中拣取对应的物料盘（Reel），并通过 **料架 LED 亮灯** 引导操作员快速定位。

---

## 二、完整操作流程

```
BOM上传 ──→ 生成发料单 ──→ FIFO计算 ──→ LED亮灯分配 ──→ 扫码取料 ──→ 完成
   ①            ②              ③              ④              ⑤         ⑥
```

---

### Step 1：上传 BOM（前置）

参见《BOM 上传操作说明》。需先上传 BOM 并生成发料单。

### Step 2：生成发料单

| PC Web 操作 |
|------------|
| BOM 管理 → 选择 BOM → "生成发料单" |
| 选择客户 → 输入（可选）需求日期 → 确认 |

系统创建 **发料单（IssueOrder）**，状态为 `pending`。

### Step 3：FIFO 计算

| 端 | 操作 |
|----|------|
| PC Web | 发料管理 → 选择发料单 → 点击"FIFO计算" |
| PDA | 出库 → 选择发料单 → 点击"计算" |

**出库策略**（在系统设置中选择）：
| 策略 | 说明 | 适用场景 |
|------|------|---------|
| `tail_first`（推荐） | 优先出库余量少的盘（尾数优先） | 减少零散盘数量 |
| `time_fifo` | 严格按入库时间先后出库 | 先进先出 |
| `mixed` | 同尾数时按时间排序 | 兼顾尾数和时间 |

**系统计算结果展示**：

```
┌─────────────────────────────────────────┐
│  物料编码       需求    可用    短缺     │
│  RES-0402-1001  100     200     0       │
│  ├─ 取盘 #1024 （余30）   取30          │
│  ├─ 取盘 #1028 （余80）   取70          │
│  CAP-0603-104   50      120     0       │
│  ├─ 取盘 #1032 （全盘）   取满          │
│  └─  ...                                │
│  短缺物料：无                           │
│  策略：tail_first                       │
│                                         │
│  [确认分配并亮灯]                        │
└─────────────────────────────────────────┘
```

### Step 4：LED 亮灯引导

点击"确认分配并亮灯"后：

| 系统行为 |
|----------|
| 1. 根据计算结果，确定需要取哪些储位上的 Reel |
| 2. 向料架发送 Modbus 指令，**点亮对应储位的 LED 灯**（绿色） |
| 3. 发料单状态变为 `assigned` |

操作员看到料架上亮起绿灯的储位，即为需要取料的位置。

> 若涉及多料架多储位，所有目标储位同时亮灯，操作员可按顺序取料。

### Step 5：扫码取料

| PDA 操作 |
|----------|
| 到亮灯储位取下 Reel → 扫描 Reel 上的条码 |

**系统校验**：
| 校验项 | 通过 | 不通过 |
|--------|------|--------|
| 条码是否属于本次发料单 | ✅ 继续 | ❌ 提示"该盘不在本次发料单中" |
| 数量是否足够 | ✅ 继续 | ⚠️ 部分消耗（取部分量） |
| 是否重复出库 | ✅ 继续 | ❌ 提示"该盘已出库" |

**校验通过后**：

```
┌───────────────────────────────┐
│  ✅ 取料成功                   │
│  物料: RES-0402-1001           │
│  本次取量: 30                   │
│  物料剩余需求: 0               │
│  LED: 已熄灭                   │
│                               │
│  该物料需求已全部出库 ✅        │
└───────────────────────────────┘
```

**系统自动执行**：
1. 减少库存盘数量（若取部分，原盘减量；若取完，原盘设 `exhausted`）
2. 记录 Transaction（type=out）
3. 熄灭该储位的 LED
4. 更新发料明细的已出库数量
5. 若所有物料全部出库完成，发料单自动标记 `completed`

### Step 6：完成发料

当发料单所有物料全部出库完成时，状态自动变为 `completed`。

操作员也可在 PDA/PC 上手动确认发料单完成。

## 三、部分取料说明

当某个物料需求数量小于整盘数量时，系统支持**部分消耗**：

```
场景：需求 30 盘，库存盘 #1024 有 50 盘
结果：盘 #1024 数量从 50 → 20（减少 30，状态保持 on_shelf）
      新产生出库记录：30 盘
```

> 部分消耗后，原盘继续留在储位上，下次出库仍会被 FIFO 算法选中。

---

## 四、短缺处理

如果库存不足，系统会标记短缺数量：

```
┌────────────────────────────────┐
│  ⚠️ 部分物料库存不足             │
│                                │
│  IC-LM358   需求 20  短缺 5     │
│                                │
│  [仅出库可用量]  [取消发料单]   │
└────────────────────────────────┘
```

操作员可选择：
- **仅出库可用量** — 有多少出多少，短缺部分后续补料
- **取消发料单** — 本次不发料，等补货后再重新计算

---

## 五、替代料出库

如果某种物料 A 库存不足，但系统中配置了替代料 B（通过 `MaterialAlternative` 表）：

| 系统行为 |
|----------|
| 1. 物料 A 库存不足时，自动查询替代料 B |
| 2. 若替代料 B 库存充足，将 B 纳入计算 |
| 3. 计算结果中标注"替代: A → B" |

## 六、数据流参考

### 6.1 涉及的表

| 表 | 操作 | 说明 |
|----|------|------|
| `issue_order` | INSERT → UPDATE | 状态流转: pending → calculating → assigned → completed |
| `issue_detail` | INSERT → UPDATE | 状态流转: pending → picking → completed |
| `inventory_pallets` | UPDATE | 出库后减少 quantity 或设为 exhausted |
| `transactions` | INSERT | type=out，记录出库流水 |
| `led_commands` | INSERT → UPDATE | 创建亮灯指令 → 执行 → 清除 |

### 6.2 状态机

```
发料单:
  pending → calculating → calculated → assigned → (取料中) → completed

发料明细:
  pending → picking → completed

库存盘 quantity:
  取部分: quantity -= pick_qty
  取完:   status = "exhausted", quantity = 0
```

## 七、异常处理

| 异常 | 系统行为 | 操作员处理 |
|------|---------|-----------|
| LED 不亮 | 系统提示 LED 失败，但仍可正常出库 | 按储位号手动找到位置，继续操作 |
| 扫码条码与发料单不匹配 | 拒绝出库 | 检查是否拿错了 Reel |
| 取料后数量不对 | 按实际扫码数量为准 | 系统自动计算部分消耗 |
| 储位上无盘但 LED 亮 | 可能是误触或数据不一致 | 在系统中清除该 LED，检查库存数据 |
| 料架通信中断 | 亮灯功能不可用，出库流程其余正常 | 检查料架网络连接 |

## 八、常见问题

**Q: 可以同一个发料单多人同时取料吗？**
A: 可以。系统支持多人并发取料，每个取料请求独立校验和更新库存。

**Q: LED 灯亮了多久？**
A: 默认常亮直到扫码取料后自动熄灭。如果长时间不取料，可在系统中手动清除 LED。

**Q: 如果取了一半物料，剩下的怎么处理？**
A: 下次同一个物料再次出库时，FIFO 算法会优先选中这个已部分消耗的盘（tail_first 策略下）。

**Q: 发料单可以部分取消吗？**
A: 当前不支持部分取消。如需调整，可完成本次发料后另建新发料单。
