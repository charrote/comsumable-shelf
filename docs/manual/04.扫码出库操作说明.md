# PDA 扫码出库操作说明

> 适用端：PDA
> 对应角色：仓库操作员
> 最后更新：2026-06-27
> 关联版本：v1.0

---

## 一、业务目标

从库存中取出物料盘（Reel）完成出库操作。PDA 端支持两种出库模式：

| 模式 | 适用场景 | 前置条件 |
|------|---------|---------|
| **按料单出库** | 按 BOM 需求批量发料至产线 | 需先上传 BOM 并生成发料单 |
| **单独出料** | 退料、废料处理、紧急出库 | 无前置条件，扫盘即出 |

---

## 二、操作入口

| 操作 | 说明 |
|------|------|
| PDA 底部导航 → **扫码出库** | 进入出库方式选择界面 |

进入后显示两个模式入口卡片，点击选择对应的出库方式：

```
┌──────────────────────────────┐
│     📄 按料单出库             │
│  选择发料单 → FIFO计算       │
│  → LED亮灯 → 扫码取料        │
├──────────────────────────────┤
│     📦 单独出料               │
│  扫描料盘条码 → 直接出库      │
│  （退料/废料/紧急）           │
└──────────────────────────────┘
```

---

## 三、按料单出库

### 业务流程

```
选择发料单  →  FIFO计算  →  LED亮灯分配  →  扫码取料  →  完成
    ①            ②             ③             ④          ⑤
```

### Step 1：选择发料单

| PDA 操作 |
|----------|
| 在出库方式选择界面点击「按料单出库」 |
| 系统加载待处理的发料单列表（状态为 `pending` / `assigned` / `picking`） |
| 点击目标发料单查看详情 |

**PDA 显示**：
```
┌──────────────────────────────────┐
│  ← 返回  按料单出库              │
│           单号: ISS-20260627001  │
├──────────────────────────────────┤
│  选择发料单                      │
│  [刷新发料单列表]                │
│                                   │
│  ┌──────────────────────────┐   │
│  │ #ISS-20260627001  [assigned] │
│  │ 产品: PCB-1001            │   │
│  │ 5 项物料 × 100 套         │   │
│  └──────────────────────────┘   │
│  ┌──────────────────────────┐   │
│  │ #ISS-20260627002  [pending]  │
│  │ 产品: PCB-1002            │   │
│  │ 8 项物料 × 50 套          │   │
│  └──────────────────────────┘   │
└──────────────────────────────────┘
```

**发料单状态说明**：

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| `pending` | 待处理，未进行 FIFO 计算 | FIFO 计算 → LED 亮灯 → 扫码取料 |
| `assigned` | 已计算分配，可继续取料 | LED 亮灯 → 扫码取料（或接续取料） |
| `picking` | 取料中，部分物料已出库 | 扫码取料（继续未完成的取料） |
| `completed` | 已全部出库完成 | 不可操作（列表中不显示） |

> 列表仅显示 `pending` / `assigned` / `picking` 状态的发料单。

---

### Step 2：FIFO 计算

选择发料单后，点击「FIFO 计算」按钮。

| 系统行为 |
|----------|
| 1. 根据当前库存，按设定的**出库策略**计算每个物料从哪些 Reel 中取料 |
| 2. 自动处理**短缺**和**替代料**逻辑 |
| 3. 计算结果中列出每个物料计划取哪些盘、各取多少数量 |

**出库策略**（在系统设置中配置）：

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| `tail_first`（推荐） | 优先出库余量少的盘（尾数优先） | 减少零散盘数量 |
| `time_fifo` | 严格按入库时间先后出库 | 先进先出 |
| `mixed` | 同尾数时按时间排序 | 兼顾尾数和时间 |

**PDA 显示 - 计算结果**：
```
┌──────────────────────────────────┐
│  FIFO 计算结果                    │
│  策略: tail_first                │
│                                   │
│  RES-0402-1001                   │
│  贴片电阻 100Ω                    │
│  需求: 100  可用: 200  已分配: 100│
│    └─ 盘 #1024  取 30  储位 A5   │
│    └─ 盘 #1028  取 70  储位 A8   │
│                                   │
│  CAP-0603-104                    │
│  贴片电容 0.1μF                   │
│  需求: 50  可用: 120  已分配: 50  │
│    └─ 盘 #1032  取 50  储位 B2   │
│                                   │
│  IC-LM358 (⚠️ 短缺 5)            │
│  需求: 20  可用: 15  已分配: 15  │
│    └─ 盘 #1040  取 15  储位 C3   │
└──────────────────────────────────┘
```

**短缺处理**：
- 如果某种物料库存不足，该物料会标注短缺数量
- 短缺物料仍会按可用量出库（有多少出多少）
- 短缺部分后续补货后可重新计算

---

### Step 3：LED 亮灯分配

确认 FIFO 计算结果后，点击「LED 亮灯」按钮。

| 系统行为 |
|----------|
| 1. 根据计算结果，向料架发送亮灯指令 |
| 2. **绿色 LED 灯**点亮所有需要取料的储位 |
| 3. 发料单状态更新为 `picking` |

**PDA 显示**：
```
已生成 5 个亮灯指令
```

操作员前往料架区域，看到亮起**绿灯**的储位即为需要取料的位置。多个储位同时亮灯时，可按任意顺序取料。

> **注意**：如果 LED 亮灯失败（料架通信异常），系统会告警但不影响出库流程，操作员可按储位号手动找到目标位置。

---

### Step 4：扫码取料

LED 亮灯后，操作员前往亮灯储位取下 Reel，使用 PDA 扫描 Reel 上的条码。

| PDA 操作 |
|----------|
| 在"扫码取料"区域扫描 Reel 条码（摄像头扫码或手动输入） |
| 确认后系统自动执行取料 |

**PDA 显示**：
```
┌──────────────────────────────────┐
│  扫码取料                        │
│  扫描亮灯储位上的料盘条码        │
│                                   │
│  [扫描输入框         📷]        │
│                                   │
│       [确认出库]                  │
│                                   │
│  ── 上次取料结果 ──              │
│  ✅ 已拣料                        │
│  已拣: 30 / 剩余: 70             │
│  已清除 1 个亮灯                  │
└──────────────────────────────────┘
```

| 系统校验 |
|----------|
| 1. 校验条码对应的 Reel 是否属于本次发料单 |
| 2. 校验取料数量是否合理 |
| 3. 检查是否重复出库 |

| 校验通过后系统自动执行 |
|------------------------|
| 1. 减少 Reel 库存数量（部分取料时原盘减量，取完时状态设为 `exhausted`） |
| 2. 记录 Transaction（type=out） |
| 3. **熄灭**该储位的 LED 灯 |
| 4. 更新发料明细的已出库数量 |
| 5. 若物料需求全部出库完成，标记该行为 `completed` |
| 6. 若所有物料全部出库完成，发料单自动标记 `completed` |

**取料结果展示**：
```
✅ 该物料需求已全部出库
```

或若该物料尚未取完：
```
已拣: 30 / 剩余: 70
已清除 1 个亮灯
```

---

### Step 5：完成发料

当发料单中所有物料全部出库完成时，系统提示"本出库单已全部拣料完成"，发料单状态自动变为 `completed`。

操作员可点击「← 返回」回到发料单列表，开始下一个发料单。

## 四、单独出料

### 业务流程

```
扫描料盘条码  →  确认出库信息  →  输入出库数量  →  确认出库
     ①                ②                ③             ④
```

### Step 1：扫描料盘

| PDA 操作 |
|----------|
| 在出库方式选择界面点击「单独出料」 |
| 扫描待出库 Reel 上的条码（摄像头扫码或手动输入） |
| 系统查询并展示料盘信息 |

**PDA 显示**：
```
┌──────────────────────────────────┐
│  ← 返回  单独出料                │
│           扫描料盘 → 确认出库    │
├──────────────────────────────────┤
│  扫描料盘                        │
│                                   │
│  [扫描输入框         📷]        │
│                                   │
│       [扫描识别]                  │
└──────────────────────────────────┘
```

### Step 2：查看料盘信息

扫描成功后，系统展示料盘的详细信息：

```
┌──────────────────────────────────┐
│  料盘信息                        │
│                                   │
│  Reel #: 1024                    │
│  物料: RES-0402-1001            │
│  贴片电阻 100Ω                   │
│  当前数量: 30                    │
│  储位: SH-A-01                   │
│                                   │
│  出库数量                         │
│  [ 30        ]                   │
│                                   │
│       [确认出库]                  │
└──────────────────────────────────┘
```

### Step 3：输入出库数量

| PDA 操作 |
|----------|
| 查看当前库存数量（默认填满） |
| 如需部分出库，可修改出库数量（不能超过当前库存） |
| 点击「确认出库」 |

> 单独出库**支持部分出库**，不必整盘出完。例如当前数量 30，可输入 10 只出 10 盘，剩余 20 盘留在库存中。

### Step 4：确认出库

| 系统执行 |
|----------|
| 1. 校验 Reel 存在且可出库 |
| 2. 减少库存数量（`quantity -= 出库数量`） |
| 3. 若数量归零，状态设为 `exhausted`，可选释放储位 |
| 4. 记录 Transaction（type=out, source_type=direct_outbound） |
| 5. 若整盘出库且配置释放储位，清除 `shelf_slot_id` |

**出库结果展示**：
```
┌──────────────────────────────────┐
│  ✅ 出库成功                      │
│                                   │
│  出库前: 30 → 出库后: 0          │
│  储位释放: 是                     │
│  出库成功，盘 #1024 数量 30 → 0  │
└──────────────────────────────────┘
```

## 五、两种出库模式对比

| 维度 | 按料单出库 | 单独出料 |
|------|-----------|---------|
| 前置条件 | 需 BOM → 发料单 → FIFO 计算 → LED 亮灯 | **无任何前置** |
| 操作步骤 | 5 步（选单→计算→亮灯→取料→完成） | **2 步**（扫 Reel → 确认） |
| 出库记录 | 关联发料单号（`issue order`） | 仅记录 Transaction |
| 库存影响 | 可部分消耗，完整跟踪 | 可部分出库，也可整盘出完 |
| LED 亮灯 | ✅ FIFO 计算后自动亮绿灯引导 | ❌ 不需要 |
| 适用场景 | 批量按 BOM 发料至产线 | 零星/异常出库（退料、废料、紧急） |

## 六、部分取料说明

按料单出库和单独出料均支持部分消耗：

| 场景 | 结果 |
|------|------|
| 需求 30，库存盘有 50 | 盘数量 50→20（减少 30，状态保持 `on_shelf`） |
| 需求 50，库存盘有 50 | 盘数量 50→0，状态变 `exhausted`，储位释放 |

> 部分消耗后原盘继续留在储位上，下次出库仍会被 FIFO 策略选中（`tail_first` 下优先选尾数少的）。

## 七、数据流参考

### 涉及 API

| API | 方法 | 说明 |
|-----|------|------|
| `/issues` | GET | 获取发料单列表 |
| `/issues/{id}` | GET | 获取发料单详情 |
| `/issues/{id}/calculate` | POST | FIFO 计算 |
| `/issues/{id}/assign` | POST | LED 亮灯分配 |
| `/issues/{id}/confirm-pick` | POST | 扫码确认取料 |
| `/inventory/scan-reel` | POST | （单独出料）扫描料盘获取信息 |
| `/inventory/reels/{id}/direct-out` | POST | （单独出料）直接出库 |

### 涉及数据库表

| 表 | 操作 | 说明 |
|----|------|------|
| `issue_order` | INSERT → UPDATE | 状态流转: `pending` → `assigned` → `picking` → `completed` |
| `issue_detail` | INSERT → UPDATE | `requirement_qty`, `picked_qty`, `status` |
| `inventory_reels` | UPDATE | `quantity -= pick_qty`，或 `status = exhausted` |
| `transactions` | INSERT | `type=out`，记录出库流水 |
| `led_commands` | INSERT → UPDATE | 创建亮灯指令 → 扫码后清除 |
| `shelf_slots` | UPDATE | （可选）释放储位时更新 |

### 状态机

```
发料单:
  pending ──→ assigned ──→ picking ──→ completed

发料明细:
  pending ──→ partial ──→ completed

库存盘 quantity:
  部分出库: quantity -= out_qty
  全部出完: status = "exhausted", quantity = 0, shelf_slot_id = null
```

## 八、异常处理

### 按料单出库

| 异常 | 系统行为 | 操作员处理 |
|------|---------|-----------|
| LED 不亮 | 系统提示 LED 失败，但不影响出库 | 按计算结果中的储位号手动找到位置 |
| 扫码条码不属于本次发料单 | 提示"该盘不在本次发料单中" | 检查是否拿错了 Reel |
| 取料后数量不对 | 按实际扫码数量为准 | 系统自动计算部分消耗，无需干预 |
| 储位上无盘但 LED 亮 | 可能是误触或数据不一致 | 在系统中清除该 LED，检查库存数据 |
| 料架通信中断 | 亮灯功能不可用，出库流程其余正常 | 检查料架网络连接 |
| 发料单已全部出完但有剩余扫码 | 提示"该物料需求已全部出库" | 检查是否重复操作 |

### 单独出料

| 异常 | 系统行为 | 操作员处理 |
|------|---------|-----------|
| Reel 已出库（exhausted） | 拒绝，提示"不可重复出库" | 检查是否重复操作 |
| Reel 状态为 tracking | 拒绝，提示"该盘正在退库跟踪中" | 先完成退库流程 |
| 出库数量超过库存 | 拒绝，提示"出库数量超过库存" | 修改出库数量 |
| Reel 不存在 | 拒绝，提示"未找到该料盘" | 检查条码是否正确 |
| 网络断开 | PDA 请求失败，提示重试 | 检查网络后重试 |

## 九、常见问题

**Q: 同一个发料单可以多人同时取料吗？**
A: 可以。系统支持多人并发取料，每个取料请求独立校验和更新库存。

**Q: LED 灯亮了多久？**
A: 默认常亮直到扫码取料后自动熄灭。如果长时间不取料，可在系统中手动清除 LED。

**Q: 如果取了一半物料，剩下的怎么处理？**
A: 下次同一物料再次出库时，FIFO 算法会优先选中已部分消耗的盘（`tail_first` 策略下）。

**Q: 单独出库后还能撤销吗？**
A: 不能直接撤销。可人工创建一个新的入库单，将物料重新入库（反操作）。

**Q: 单独出库会影响 FIFO 计算吗？**
A: 不会。FIFO 计算时自动过滤 `status = exhausted` 的盘。

**Q: 单独出库需要什么权限？**
A: 建议限制为 `supervisor` 及以上角色操作，避免误操作导致库存丢失。

**Q: 发料单可以部分取消吗？**
A: 当前不支持部分取消。如需调整，可完成本次发料后另建新发料单。

**Q: PDA 端和 PC Web 端的区别是什么？**
A: PDA 端面向现场操作，聚焦扫码执行，流程简洁；PC Web 端面向管理，支持 BOM 上传、发料单创建、FIFO 计算查看、库存查询等管理功能。
