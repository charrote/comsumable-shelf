# 收料操作说明

> 适用端：PDA（主） / PC Web
> 对应角色：仓库操作员
> 最后更新：2026-06-17
> 关联版本：v1.0

---

## 一、业务目标

将**客户来料**录入系统库存，使其成为可追溯的库存盘（Reel），并为后续上架做准备。

收料环节完成三件事：
1. **识别物料** — 通过扫描客户标签，自动或人工确认物料编码
2. **创建库存** — 生成系统库存盘记录（InventoryReel）
3. **生成内部标签** — 打印含内部料号、客户料号、Reel编码、数量的标签贴于盘上

---

## 二、操作流程

### 2.1 前置条件

| 条件 | 说明 |
|------|------|
| 已创建入库单 | 操作员在 PDA/PC 上创建入库单（自动编号 RC-YYYYMMDD-NNN） |
| 客户标签完好 | 来料外箱/盘上附有条形码或二维码标签 |
| 打印机已连接 | 标签打印机网络可达（IP + Port 已配置） |

### 2.2 分步操作

#### Step 1：创建入库单

| 端 | 操作 |
|----|------|
| PDA | 首页 → 入库 → 点击"新建入库单" → 输入操作员姓名 → 确认 |
| PC Web | 入库管理 → 创建入库单 → 输入操作员姓名 → 确认 |

系统自动生成入库单号 `RC-YYYYMMDD-001`（每日从 001 开始顺序编号）。

#### Step 2：扫码收料（核心步骤）

使用 PDA 扫描枪或 PC 条码输入框扫描**客户标签**上的条码。

系统根据条码内容自动执行物料匹配，分三种场景：

---

##### 场景 A：自动匹配成功 ✅

**条件**：条码解析后与系统中已有物料编码精确匹配，或模糊匹配置信度 ≥ 60%。

**系统行为**：
1. 自动创建库存盘（InventoryReel），状态=`on_shelf`
2. 自动分配空储位（若存在）
3. 自动触发标签打印
4. 返回结果卡片

**PDA/PC 显示**：
```
┌─────────────────────────┐
│  状态: ✅ 成功            │
│  条码: 2024ABC-12345     │
│  物料编码: RES-0402-1001 │
│  物料名称: 贴片电阻 100Ω  │
│  库存盘 #: 1024           │
│  分配储位: SH-A-01-05     │
│  数量: 1 盘               │
│  标签: 已打印             │
└─────────────────────────┘
```

**操作员**：将打印出的标签贴到盘上 → 继续下一盘 → 或点击"完成入库"。

---

##### 场景 B：匹配置信度低，需人工选择 ⚠️

**触发条件**：条码解析后无精确匹配，模糊匹配置信度在 30%~60% 之间。

**系统行为**：
1. 返回候选物料列表（TOP 5，按置信度降序）
2. 弹出物料确认对话框

**PDA/PC 显示**：
```
┌──────────────────────────────────┐
│  ⚠️ 物料确认                      │
│  客户条码: 2024ABC-12345          │
│  识别编码: 2024ABC-12345          │
│                                   │
│  ○ 选择已有物料                    │
│  ┌────────────────────────────┐   │
│  │ RES-0402-1001  贴片电阻 90% │   │
│  │ CAP-0603-104  贴片电容 72%  │   │
│  │ RES-0402-1002  贴片电阻 55% │   │
│  └────────────────────────────┘   │
│                                   │
│  [确认为新料]                      │
└──────────────────────────────────┘
```

**操作员**：
- **如果有匹配的物料** → 点击对应的物料行 → 确认入库
- **如果无匹配** → 点击"确认为新料" → 输入物料编码和名称（可选）→ 确认

> 确认为新料后，系统自动创建 `MaterialMaster` 记录，
> 物料编码默认使用条码中提取的编码，操作员可修改。

---

##### 场景 C：完全不匹配，作为新料入库 🆕

**触发条件**：条码解析后置信度 < 30%，无任何候选。

**系统行为**：
1. 弹出"确认为新料"对话框
2. 自动填入从条码提取的编码作为物料编码

**操作员**：
- 确认使用提取的编码 → 点击确认
- 或修改物料编码/输入物料名称 → 点击确认

> 系统自动创建 `MaterialMaster`，unit 默认为"盘"。

---

##### 场景 D：重复扫码 🔄

**触发条件**：扫描的条码已在库存中存在（status = `on_shelf` 或 `tracking`）。

**系统行为**：
- 直接拦截
- 显示已有库存盘信息

**操作员**：检查是否重复操作 → 如需强制入库需主管处理。

---

#### Step 3：完成入库

扫码完成后，点击"完成入库"按钮。

| 操作 | 说明 |
|------|------|
| 确认入库单 | 系统将入库单状态从 `draft` → `confirmed`（后续还可继续补扫） |
| 完成入库单 | 入库单状态从 `confirmed` → `completed`（不可再扫） |

## 三、数据流参考

### 3.1 涉及的表

| 表 | 操作 | 说明 |
|----|------|------|
| `receipt` | INSERT | 入库单头（自动编号 RC-YYYYMMDD-NNN） |
| `receipt_pallets` | INSERT | 每盘收料记录 |
| `inventory_pallets` | INSERT | 库存盘记录 |
| `inventory_pallets.shelf_slot_id` | UPDATE | 自动分配储位时写入 |
| `material_master` | INSERT | 仅当确认为新料时自动创建 |
| `transactions` | INSERT | 自动记录入库交易流水（type=in） |

### 3.2 状态机

```
入库单状态:
  draft ──→ confirmed ──→ completed
     ↑            │
     └────────────┘ (可多次补扫确认)

库存盘状态:
  创建时默认为 on_shelf
```

## 四、异常处理

| 异常 | 系统行为 | 操作员处理 |
|------|---------|-----------|
| 打印机离线 | 入库成功，标签标记未打印 | 稍后在入库明细中点击"重打" |
| 条码无效 | 返回"无效的条码格式" | 检查条码是否完整，重扫 |
| 储位已满 | 入库成功，不分配储位 | 后续在上架环节手动分配 |
| 网络断开 | PDA 本地缓存，上线后自动提交 | 确认缓存队列正常 |
| 重复扫同一盘 | 拦截 + 提示已有记录 | 确认是否重复操作 |

## 五、常见问题

**Q: 为什么扫了条码提示"物料不存在"？**
A: 这是旧版本行为。当前版本会直接弹出"确认为新料"对话框，不会报错退出。

**Q: 标签没打印出来怎么办？**
A: 检查打印机 IP/Port 是否在系统设置中正确配置。稍后可在入库明细中手动点击"重打标签"。

**Q: 扫码时选错了物料怎么办？**
A: 当前不支持直接撤销。可在库存管理中删除刚创建的库存盘（需管理员权限），然后重新扫码。

**Q: 一个入库单可以扫多少盘？**
A: 不限。可在"完成入库"前持续扫码，所有扫码记录归入同一入库单。
