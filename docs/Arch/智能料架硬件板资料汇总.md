### 3.2 硬件接口

| 接口 | 引脚定义 | 说明 |
|------|----------|------|
| **JP1 (RS485+电源)** | A+, B-, VDD, VDD, GND, GND | RS485 通信 + 8-16V 供电，双 VDD+GND 增流 |
| **JP2 (RS485+电源)** | 同 JP1 | 内部并联，同 JP1 |
| **JP3 (烧录+电源)** | TXD, RXD, VDD, VDD, GND, GND | TTL 电平烧录 + 8-16V 供电 |

> JP1/JP2 内部并联；JP3 的 VDD/GND 也与 JP1/JP2 并联。

### 3.3 按键操作

| MODE 按键次数 | 模式 | 数码管显示 | FUN 按键功能 |
|---------------|------|-----------|-------------|
| 首次上电 | 正常工作 | "188" → 地址 | 启动校准 |
| 1 次 | 配置地址 | "F1" → 地址 | 点按+1, 长按0.3s+1, 最大"40"归零 |
| 2 次 | 配置算法 | "F2" → U1/U2 | 切换 U1/U2 算法模式 (仅 CH20 有效) |
| 3 次 | 厂家测试 | "F3" → P1~P9 | 红绿蓝交替闪烁，P1=储位1异常 |
| 4 次 | 设置LED模式 | "F4" → 0~5 | 切换储位变化后LED显示模式 |
| 5 次 | 存储配置 | "F0" | 启动校准并存储 |
| 长按3秒 | 灯板复位 | 同首次上电 | — |

**LED 显示模式 (F4 模式)**:
- 模式 0: 储位变化, LED 绿灯常亮
- 模式 1: 储位变化, LED 绿灯亮 1 秒后恢复
- 模式 2: 储位变化, LED 蓝灯常亮
- 模式 3: 储位变化, LED 蓝灯亮 1 秒后恢复
- 模式 4: 储位变化, LED 红灯常亮
- 模式 5: 储位变化, LED 红灯亮 1 秒后恢复

**校准**: 正常工作模式下按 FUN → 绿灯从储位1到X依次亮灭 → 校准完成
- 不亮灯 = 检测 OK
- 绿灯 = 有偏差但可用
- 蓝灯 = 偏差较大，需维修
- 红灯 = 器件故障

### 3.5 LED 控制详解

#### 协议1: 地址 0-59 (功能码 1,15)

每个 LED 占 1 个线圈地址，格式: LEDnG/G/B = n, n+1, n+2

```
LED1G=0x0000, LED1R=0x0001, LED1B=0x0002
LED2G=0x0003, LED2R=0x0004, LED2B=0x0005
...
LED20G=0x0039, LED20R=0x003A, LED20B=0x003B
```

#### 协议2: 地址 10000+ (推荐, 功能码 15)

每 LED 占 4 个线圈地址: G(亮灭) + R(亮灭) + B(亮灭) + 控制位(有效)

```
LED1:  10000(G) 10001(R) 10002(B) 10003(控制位)
LED2:  10004(G) 10005(R) 10006(B) 10007(控制位)
...
LED20: 10076(G) 10077(R) 10078(B) 10079(控制位)
```

**字节映射 (写多线圈)**:
- Bit0~2 = LED1 的 G/R/B；Bit3 = LED1 控制位
- Bit4~6 = LED2 的 G/R/B；Bit7 = LED2 控制位
- Bit0~2 = LED3 的 G/R/B；Bit3 = LED3 控制位
- ... 以此类推，每 3 bit 对应一个颜色

### 3.6 储位信息读取

#### 数字量输入寄存器 (0-19, 功能码 2, 只读)

| 地址 | 含义 |
|------|------|
| 0x0000 | 储位1: 1=有货物, 0=空 |
| 0x0001 | 储位2 |
| ... | ... |
| 0x0013 | 储位20 |

**读取示例** (灯板地址=1):
```
请求: 01 02 00 00 00 14 78 05    (读 20 个线圈)
应答: 01 02 03 12 80 01 78 4B
  字节1=0x12=0001_0010 → 储位5,2 有货物
  字节2=0x80=1000_0000 → 储位16 有货物
  字节3=0x01=0000_0001 → 储位17 有货物
```

### 3.7 灯板信息寄存器 (地址 40-44, 功能码 4, 只读)

| 地址 | 含义 |
|------|------|
| 40 | 灯板型号: 高字节 0x43='C', 低字节 0x14=20路, 0x0A=10路, 0x06=6路 |
| 41 | 硬件版本: 如 0x0073=V1.15 |
| 42 | 软件版本: 如 0x0073=V1.15 |
| 43 | 编译日期年: 如 0x2022=2022年 |
| 44 | 编译日期日月: 如 0x0716=7月16日 |

### 3.8 灯板控制命令 (保持寄存器 0-10, 功能码 3/6/16)

| 地址 | 功能 | 说明 |
|------|------|------|
| 0x0000 | 控制命令 | 0x5AA5=复位, 0x5AA6=校准, 0x5AA8=设判定值, 0x5AA9=逐位设判定值 |
| 0x0001~0x000A | 判定值 | 地址1高字节=储位1判定值, 地址1低字节=储位2判定值, 以此类推 |

**判定值**: 范围 0-255, 单位 0.0196V。如 0.8V → 41 (0.8/0.0196≈41)。
- 20通道: 一般 0.6-1.2V
- 14/12/10/6通道: 一般 1.2-2.2V

**校准命令**:
```
设判定值=0x29(0.8V):  01 10 00 00 00 02 04 5A A8 00 29 A1 49
执行校准:              01 06 00 00 5A A6 33 10
设备复位:              01 06 00 00 5A A5 73 11
```

### 4.1 数据流

```

  │
  │ Modbus TCP (端口 502)
  ▼
┌──────────────────┐
│ AMKN8702G 主控板 │  站号: 200
│                  │  功能:
│  · 继电器控制     │  1. Modbus TCP 解析
│  · 协议转换       │  2. Modbus TCP → RTU 转换
│  · 数据采集聚合   │  3. 储位状态缓存
│  · 固件 OTA       │  4. WiFi/Ethernet 切换
│  · 语音控制       │  5. 配置管理
└────────┬─────────┘
         │ Modbus RTU
         │ 38400bps, 8N1
         │ A面: COM1 (站号 1-63)
         │ B面: COM2 (站号 1-63)
         ▼
┌──────────────────┐
│ AMKN7141 灯板 ×N │  地址 1-N
│                  │  功能:
│  · 红外检测       │  1. AD 采样储位检测

│  · 判定值配置     │  4. 本地 LED 逻辑
└──────────────────┘
```

### 4.2 关键协议对照

| 层级 | 协议 | 地址 | 端口/波特率 | 功能码 |
|------|------|------|------------|--------|
| PC↔主控板 | Modbus TCP | 主控=200, A面=1-63, B面=64-126 | 502 | 1,2,5,6,15,16 |
| 主控板↔灯板 | Modbus RTU | 灯板 1-N (每面) | 38400bps, 8N1 | 2,4,6,15,16 |
| 灯板内部 | — | 每 LED 3 线圈 (协议1) | — | — |
| | | 每 LED 4 线圈 (协议2) | — | — |

### 4.3 储位地址映射 (主控板→灯板)

A面灯板 i (i=1..N), 每板 M 储位:
- 主控板数字量输入起始地址 = 1000 + (i-1)×M
- B面起始地址 = 2000 + (i-1)×M
