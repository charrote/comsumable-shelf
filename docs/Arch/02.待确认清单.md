# 需客户确认/提供的交付物清单

为确保开发顺利推进，需向客户明确索取以下资料与确认项：

## 1. 硬件通讯协议
| 设备名称       | 需提供内容                             | 影响范围        |
| :--------- | :-------------------------------- | :---------- |
| **智能料架**   | 控制 SDK / API 文档（含亮灯指令集、状态回调）      | 硬件驱动开发、数据同步 |
| **XR 点料机** | 数据输出格式、通讯方式（蓝牙/串口/WIFI）、对接协议或 SDK | 硬件驱动开发、数据同步 |

## 2. 基础数据模板
| 资料名称      | 需提供内容                    | 影响范围           |
| :-------- | :----------------------- | :------------- |
| **BOM 表** | 标准模板（Excel/CSV）          | BOM 解析算法、数据初始化 |
| **替代料关系** | 1:1 或 1:N？是否含换算比例或优先级规则？ | BOM 解析算法、数据初始化 |
| **物料字典**  | 编码规则、客户别分类、计量单位等基础字典     | BOM 解析算法、数据初始化 |

## 3. 业务规则确认
| 确认项         | 需确认内容                                       | 影响范围         |
| :---------- | :------------------------------------------ | :----------- |
| **FIFO 策略** | 主用“尾数优先 (A)”还是“时间优先 (B)”？还是两者共存 selectable？ | 核心算法、异常流程设计  |
| **防呆阈值**    | 重复扫码是**直接拦截提示**，还是允许**主管扫码授权放行**？           | 核心算法、异常流程设计  |
| **XR 配对逻辑** | `±1~5秒` 的匹配窗口是否需支持人工微调？                     | 核心算法、异常流程设计  |
