## 1. 系统架构

```
┌─────────────────────────────────────────────────────┐
│  PC / 上位机软件                                     │
│  (Python / C# / Java 等)                             │
└──────────┬──────────────────────────────────────────┘
           │ Modbus TCP (端口 502)
           │ 以太网 或  WiFi
           ▼
┌─────────────────────────────────────────────────────┐
│  AMKN8702G 工业通信板（网关/主控）                     │
│  ┌───────────────────────────────────────────────┐  │
│  │  GD32F307VE · 120MHz · 512KB Flash · 96KB RAM │  │
│  │  以太网 RJ45 · WiFi 802.11 · RS485×2 隔离     │  │
│  │  6× 继电器 (K1-K6) · RS232×2 (调试/LCD)       │  │

              │ RS485 (波特率 38400bps, 8N1)
              │  COM1 → A面灯板 (站号 1~63)
              │  COM2 → B面灯板 (站号 1~63)
              ▼
┌─────────────────────────────────────────────────────┐
│  AMKN7141-CHXX 有感应灯板 ×N                          │
│  ┌───────────────────────────────────────────────┐  │
│  │  STC8H4K32TL · 高速 AD 采样 (800KHz)           │  │
│  │  红/绿/蓝 三色 LED ×储位数                      │  │
│  │  IR 红外传感器检测储位物料状态                   │  │
│  │  MODE/FUN 按键 · 2位半数码管                    │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

### 协议层结构

| 层级 | 协议 | 说明 |
|------|------|------|
| PC ↔ 主控板 | **Modbus TCP** (端口 502) | 以太网/WiFi，TCP 长连接 |
| 主控板 ↔ 灯板 | **Modbus RTU** (RS485, 38400bps) | 主控板自动 TCP↔RTU 协议转换 |

### 站号（Slave ID）映射

| Modbus TCP 站号 | 物理目标 | 说明 |
|-----------------|----------|------|
| **200** | 主控板自身 | 访问主控板配置寄存器、继电器等 |
| **1 ~ 63** | COM1 (A面) 灯板 | 直接转发 Modbus RTU 至 COM1 |
| **64 ~ 126** | COM2 (B面) 灯板 | 主控板自动减 63，转发至 COM2 |

> **注意**：每个面灯板的物理地址必须从 1 开始连续编号，范围 1~63。
> B面访问时，TCP 站号 64 → RTU 站号 1，TCP 站号 126 → RTU 站号 63。

### 全局约定

- 所有数据为 **16 进制表示**
- 多字节数据：**高字节在前**（Big-Endian）
- Modbus RTU CRC16：多项式 `0xA001`，标准 Modbus CRC16

## 2. AMKN8702G Modbus TCP 协议

### 2.1 寄存器类型总览

> 访问地址：站号 200，端口 502

| 寄存器类型 | 地址范围 | 数量 | 功能码 | 访问方向 | 说明 |
|-----------|----------|------|--------|----------|------|
| **线圈** | `0x0000 ~ 0x0005` | 6 | 1, 5, 15 | 读/写 | 继电器 K1~K6 输出 |
| | `0x0000 ~ 0x03E7` | 1000 | — | — | 保留 |
| **数字量输入** | `0x03E8 ~ 0x07CF` | 1000 | 2 | 只读 | A面灯板储位状态 (1000~1999) |
| | `0x07D0 ~ 0x0BB7` | 1000 | 2 | 只读 | B面灯板储位状态 (2000~2999) |
| **保持寄存器** | `60000 ~ 65535` | 5536 | 3, 6, 16 | 读/写 | 配置寄存器 (固定基地址) |

### 2.2 继电器控制

**地址映射**：

| 继电器 | Modbus TCP 地址 | 功能码 5 (单线圈) | 功能码 15 (多线圈) |
|--------|-----------------|-------------------|-------------------|
| K1 | `0x0000` | `00 05 00 00 00 FF C8 05` | 字节 Bit 0 |
| K2 | `0x0001` | `00 05 00 01 00 FF C8 05` | 字节 Bit 1 |
| K3 | `0x0002` | `00 05 00 02 00 FF C8 05` | 字节 Bit 2 |
| K4 | `0x0003` | `00 05 00 03 00 FF C8 05` | 字节 Bit 3 |
| K5 | `0x0004` | `00 05 00 04 00 FF C8 05` | 字节 Bit 4 |
| K6 | `0x0005` | `00 05 00 05 00 FF C8 05` | 字节 Bit 5 |

**功能码 5 — 单继电器控制**：
```
请求: [事务ID] [协议ID=00 00] [长度=00 06] [站号=C8=200] [FC=05] [地址] [数据(FF 00=ON, 00 00=OFF)]
应答: 完整回显请求报文
```

**功能码 15 — 多继电器批量控制**：
```
请求: [事务ID] [00 00] [00 08] [C8] [0F] [起始地址=00 00] [地址数量=00 06] [字节数=01] [数据字节]
  数据字节: Bit0=K1, Bit1=K2, Bit2=K3, Bit3=K4, Bit4=K5, Bit5=K6
  FF FF FF FF FF FF → 全部 ON
  12 (二进制 0001_0010) → K5 和 K2 ON，其余 OFF
应答: [事务ID] [00 00] [00 06] [C8] [0F] [起始地址=00 00] [地址数量=00 06]
```

**功能码 1 — 读取继电器状态**：
```
请求: [00 06] [00 00] [00 06] [C8] [01] [00 00] [00 06]
应答: [00 06] [00 00] [00 04] [C8] [01] [01] [数据字节]
  数据字节: 同上，Bit0=K1...Bit5=K6
```

### 2.3 储位状态读取

**前提条件**：必须在配置寄存器中设置：
- 寄存器 60107 (物料架类型) = 1 (有感应物料架)
- 寄存器 60108 (A面灯板数量) = N₁
- 寄存器 60109 (B面灯板数量) = N₂

主控板会自动轮询所有灯板，将储位状态缓存到数字量输入寄存器中。

**地址计算**（每块灯板支持最多 20 个储位，每个储位占 1 个线圈地址）：

```
A面灯板 i (i=1..N₁)，每板 M 个储位：
  起始地址 = 1000 + (i-1) × M
  范围 = 起始地址 ~ 起始地址 + M-1

B面灯板 i (i=1..N₂)，每板 M 个储位：
  起始地址 = 2000 + (i-1) × M
  范围 = 起始地址 ~ 起始地址 + M-1
```

**批量读取示例**（假设 A面 35 块 × 20 储位 = 700 个储位）：
```
请求: [00 06] [00 00] [00 06] [C8] [02] [03 E8] [03 48]
                        ↑                      ↑
                   FC=02(读线圈)              地址数量=840(0x0348)
                   起始地址=0x03E8=1000
应答: [00 06] [00 00] [00 6C] [C8] [02] [字节数=0x69=105] [105字节数据]...
                        ↑
                      数据长度=105 字节
                        每块灯板 3 字节 (20储位 = 20 bits = 3 bytes)
                        35 块 × 3 = 105 字节
```

**字节位映射**：

| 字节 | 对应灯板 | Bit7~Bit0 | 储位 |
|------|---------|-----------|------|
| 第 1 字节 | 灯板 1 | Bit7=储位8 ... Bit0=储位1 | 1~8 |
| 第 2 字节 | 灯板 1 | Bit7=储位16 ... Bit0=储位9 | 9~16 |
| 第 3 字节 | 灯板 1 | Bit7~Bit4=保留, Bit3=储位20 ... Bit0=储位17 | 17~20 |
| 第 4 字节 | 灯板 2 | Bit7=储位8 ... Bit0=储位1 | 1~8 |
| ... | ... | ... | ... |
| 第 103 字节 | 灯板 35 | Bit7=储位8 ... Bit0=储位1 | 1~8 |
| 第 104 字节 | 灯板 35 | Bit7=储位16 ... Bit0=储位9 | 9~16 |
| 第 105 字节 | 灯板 35 | Bit7~Bit4=保留, Bit3=储位20 ... Bit0=储位17 | 17~20 |

**数据解析**：`1` = 有物料，`0` = 空位

**B面读取示例**：
```
请求: [00 06] [00 00] [00 06] [C8] [02] [07 D0] [03 48]
                          ↑
                       起始地址=0x07D0=2000 (B面)
```

### 2.5 Modbus TCP 访问示例

**示例 1 — 读取主控板基本信息**：
```
请求: 00 06 00 00 00 06 C8 03 00 00 00 03
  FC=03, 起始地址=0, 数量=3
  读取 60000, 60001, 60002
应答: 00 06 00 00 00 0A C8 03 06 XX XX XX XX XX XX
  3 个寄存器 × 2 字节 = 6 字节数据
```

**示例 2 — 设置 A 面灯板数量**：
```
请求: 00 01 00 00 00 02 C8 06 01 4C 00 0A
  FC=06, 地址=60108(0x014C), 值=10 (A面 10 块灯板)
应答: 回显请求
然后: 写入 60121=0x0001 存储参数
然后: 写入 60121=0x0005 复位生效
```

**示例 3 — 设置 WiFi AP**：
```
请求: 00 01 00 00 00 10 C8 10 01 6F 00 10 14 5A 4C 51 4C 59 00 00...00
                    ↑                    ↑                    ↑
                  FC=16(批量写)       16个寄存器, 起始60071(AP名称)  "ZQLY"
  AP名称: "ZQLY" → 0x5A51 0x4C59 0x0000 ...
然后: 写入 WiFi 密码到 60087~60102
然后: 写入 60121=0x0001 存储
然后: 拨码 S1=ON, S2=OFF (WiFi 模式) 后重启生效
```

### 2.6 主控板直连灯板寄存器（协议转换后）

当 TCP 站号为 1~63 或 64~126 时，主控板将请求 **透明转发** 为 Modbus RTU。
灯板的寄存器定义与 [第3节](#3-amkn7141-chxx-modbus-rtu-协议) 相同，但站号映射：

- TCP 站号 N (1~63) → RTU 站号 N → COM1
- TCP 站号 N (64~126) → RTU 站号 N-63 → COM2

## 4. LED 三色灯控制

### 4.1 协议 1 — 地址 0~59 (功能码 1/15)

每个 LED 占 **1 个线圈地址**，格式：`LEDnG = 3×(n-1)`, `LEDnR = 3×(n-1)+1`, `LEDnB = 3×(n-1)+2`

| 端口 | 名称 | 地址(HEX) | 地址(DEC) | 说明 |
|------|------|-----------|-----------|------|
| LED1G | 1号储位绿灯 | 0x0000 | 0 | 1=亮, 0=灭 |
| LED1R | 1号储位红灯 | 0x0001 | 1 | |
| LED1B | 1号储位蓝灯 | 0x0002 | 2 | |
| LED2G | 2号储位绿灯 | 0x0003 | 3 | |
| LED2R | 2号储位红灯 | 0x0004 | 4 | |
| LED2B | 2号储位蓝灯 | 0x0005 | 5 | |
| LED3G | 3号储位绿灯 | 0x0006 | 6 | |
| ... | ... | ... | ... | |
| LED20G | 20号储位绿灯 | 0x0039 | 57 | |
| LED20R | 20号储位红灯 | 0x003A | 58 | |
| LED20B | 20号储位蓝灯 | 0x003B | 59 | |

**功能码 15 — 批量写入（60 个线圈）**：
```
请求: [站号] [0F] [00 00] [00 3C] [08] [数据字节...] [CRC]
         ↑      ↑      ↑      ↑
        FC=15  起始0   60个   8字节数
```

**8 字节位映射**（共 20 个 LED × 3 色 = 60 线圈 = 8 字节）：

| 字节 | Bit7 | Bit6 | Bit5 | Bit4 | Bit3 | Bit2 | Bit1 | Bit0 |
|------|------|------|------|------|------|------|------|------|
| **Byte 1** | LED3R | LED3G | LED2B | LED2R | LED2G | LED1B | LED1R | LED1G |
| **Byte 2** | LED6G | LED5B | LED5R | LED5G | LED4B | LED4R | LED4G | LED3B |
| **Byte 3** | LED8B | LED8R | LED8G | LED7B | LED7R | LED7G | LED6B | LED6R |
| **Byte 4** | LED11G | LED11R | LED10B | LED10R | LED10G | LED9B | LED9R | LED9G |
| **Byte 5** | LED14B | LED13B | LED13R | LED13G | LED12B | LED12R | LED12G | LED11B |
| **Byte 6** | LED16G | LED15B | LED15R | LED15G | LED14R | LED14B | LED14G | LED13B |
| **Byte 7** | LED19G | LED19R | LED18B | LED18R | LED18G | LED17B | LED17R | LED17G |
| **Byte 8** | LED20B | LED20R | LED20G | 保留 | 保留 | 保留 | 保留 | 保留 |

> **协议 1 特点**：每个颜色独立一个 bit，无法"保持其余不变"。写入时所有 60 个线圈都会被覆盖。

**示例 — 控制 LED1G, LED3G, LED4G, LED6G 亮**：
```
Byte1 = 0x41 = 0100_0001  → LED3G(Bit6), LED1G(Bit0)
Byte2 = 0x82 = 1000_0010  → LED6G(Bit7), LED4G(Bit1)
字节其余 = 0

请求: 01 0F 00 00 00 3C 08 41 82 00 00 00 00 00 00 [CRC]
```

### 4.2 协议 2 — 地址 10000+ (功能码 15，推荐)

每个 LED 占 **4 个线圈地址**：3个颜色 + 1个控制位

| 端口 | 名称 | 地址(HEX) | 地址(DEC) | 说明 |
|------|------|-----------|-----------|------|
| LEDnG | n号绿灯 | 0x2710 + 4×(n-1) | 10000 + 4×(n-1) | 亮灭 |
| LEDnR | n号红灯 | 0x2711 + 4×(n-1) | 10001 + 4×(n-1) | 亮灭 |
| LEDnB | n号蓝灯 | 0x2712 + 4×(n-1) | 10002 + 4×(n-1) | 亮灭 |
| LEDn控制位 | n号控制位 | 0x2713 + 4×(n-1) | 10003 + 4×(n-1) | 有效/无效 |

**核心机制 — 控制位**：

| 控制位值 | 含义 |
|----------|------|
| **1** | 操作本 LED 的 Bit0~2 (G/R/B) **有效** |
| **0** | 操作本 LED 的 Bit0~2 **无效**（保持原状态） |

> **协议 2 的核心优势**：通过控制位可以实现 **局部更新** — 只修改需要的 LED，其余保持不变。

**字节位映射（每 2 个 LED = 1 字节）**：

| 字节 | Bit7 | Bit6 | Bit5 | Bit4 | Bit3 | Bit2 | Bit1 | Bit0 |
|------|------|------|------|------|------|------|------|------|
| **Byte 1** | LED2控制位 | LED2B | LED2R | LED2G | LED1控制位 | LED1B | LED1R | LED1G |
| **Byte 2** | LED4控制位 | LED4B | LED4R | LED4G | LED3控制位 | LED3B | LED3R | LED3G |
| **Byte 3** | LED6控制位 | LED6B | LED6R | LED6G | LED5控制位 | LED5B | LED5R | LED5G |
| **Byte 4** | LED8控制位 | LED8B | LED8R | LED8G | LED7控制位 | LED7B | LED7R | LED7G |
| ... | ... | ... | ... | ... | ... | ... | ... | ... |
| **Byte 40** | LED98控制位 | LED98B | LED98R | LED98G | LED97控制位 | LED97B | LED97R | LED97G |
| **Byte 41** | LED100控制位 | LED100B | LED100R | LED100G | LED99控制位 | LED99B | LED99R | LED99G |

**对于 20 路灯板**：使用 80 个线圈 (地址 10000~10079)，需 40 字节。
**对于 100 路灯板**：使用 400 个线圈 (地址 10000~10399)，需 200 字节。

**局部更新示例 — 只点亮 LED1G，其余不变**：
```
Byte1 = 0xA9 = 1010_1001
         ↑        ↑         ↑         ↑
       Bit7=1   Bit3=1    Bit1=R(灭)  Bit0=G(亮)
       LED2控   LED1控    LED2R(0)    LED1G(1)
       有效      有效
Bit2(B)=0 → LED1B 灭, Bit4~6 = 0 → LED2G/R/B 灭

控制位=0 的 LED：其 Bit0~2 被忽略，保持原状态不变
控制位=1 的 LED：其 Bit0~2 生效

请求: 01 0F 27 10 00 50 0A A9 00 00 00 00 00 00 00 00 00 [CRC]
         ↑    ↑    ↑    ↑    ↑
        FC   起始  80个  10字节数  第1字节A9 = 只改LED1G
```

### 4.3 控制位掩码计算方法

如需设置第 n 个 LED 的绿灯为 ON：

```
Byte索引 = (n-1) ÷ 2        # 0-based 字节索引
LED索引 = (n-1) % 2          # 0 = 奇数LED, 1 = 偶数LED

if LED索引 == 0 (奇数LED):
    控制位Bit = 3
    GBit = 0
    RBit = 1
    BBit = 2
else (偶数LED):
    控制位Bit = 7
    GBit = 4
    RBit = 5
    BBit = 6

设置 Byte[Byte索引]:
  Bit[控制位Bit] = 1       # 控制有效
  Bit[GBit] = 1            # 绿灯亮
  Bit[RBit] = 0            # 红灯灭
  Bit[BBit] = 0            # 蓝灯灭
  其他Bit = 0              # 其他LED控制无效
```

## 6. 配置与操作命令

### 6.1 保持寄存器 (0x0000~0x000A，功能码 3/6/16)

| 地址 | 功能 | 读/写 | 说明 |
|------|------|-------|------|
| 0x0000 | 控制命令 | 写 | 见命令表 |
| 0x0001~0x000A | 判定值 (1~20) | 写 | 配合命令使用 |

**控制命令表**：

| 命令值 | 含义 | 判定值地址含义 |
|--------|------|---------------|
| **0x5AA5** | 灯板复位 | 无 |
| **0x5AA6** | 有感应灯板校准 | 无 |
| **0x5AA8** | 统一设判定值 | 地址1低字节 = 统一判定值 |
| **0x5AA9** | 逐位设判定值 | 地址1~10分高低字节 = 储位1~20判定值 |

**判定值转换公式**：
```
判定值 = 电压(V) ÷ 0.0196  (四舍五入)
电压 = 判定值 × 0.0196 V

典型范围：
  20通道：0.6~1.2 V → 判定值 31~62
  14/12/10/6通道：1.2~2.2 V → 判定值 62~113
```

**TFTP 方式**：
1. 拨码 S1/S2 = OFF (配置模式)

**Modbus TCP 方式**：

3. 写入 60123 = 数据包长度
4. 写入 60121 = 0x0002 (存储固件)

### 主控板 — 继电器控制

```python
import socket

def send_modbus_tcp(ip, port, slave, fc, addr, payload=b""):
    """发送 Modbus TCP 请求"""
    # MBAP Header: Transaction(2) + Protocol(2) + Length(2) + Unit(1)
    length = 6 + len(payload)
    header = b'\x00\x01'  # transaction=1, protocol=0
    header += length.to_bytes(2, 'big')
    header += bytes([slave])
    
    pdu = bytes([fc]) + addr.to_bytes(2, 'big') + payload
    request = header + pdu
    # CRC16 for Modbus TCP not needed (TCP handles integrity)
    
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
        s.connect((ip, port))
        s.sendall(request)
        return s.recv(1024)

# 设置 K1=ON
resp = send_modbus_tcp('192.168.1.100', 502, 0xC8, 0x05, 0x0000, b'\xff\x00')

# 批量控制 K2,K5=ON
resp = send_modbus_tcp('192.168.1.100', 502, 0xC8, 0x0F, 0x0000,
                       b'\x00\x06\x01\x12')  # 0x12 = 0001_0010
```

### 主控板 — 读取 A面储位状态

```python
def read_storage_status(ip, face='A', count=700):
    """读取主控板储位状态
    A面起始地址: 0x03E8 (1000)
    B面起始地址: 0x07D0 (2000)
    """
    base_addr = 0x03E8 if face == 'A' else 0x07D0
    resp = send_modbus_tcp(ip, 502, 0xC8, 0x02, base_addr,
                           count.to_bytes(2, 'big'))
    
    # 解析响应
    # Skip MBAP header (7 bytes) + FC (1) + byte count (1)
    data = resp[10:]  # 数据字节
    statuses = []
    for byte_val in data:
        for bit in range(8):
            statuses.append((byte_val >> bit) & 1)
    
    return statuses  # True=有物料, False=空
```

### 灯板 — LED 控制 (协议 2)

```python
import struct

def build_led_protocol2_byte(led_num, green=True, red=False, blue=False, control=True):
    """构建协议2的单个 LED 字节
    led_num: 1~20
    返回 2 个字节: (颜色字节, 控制位)
    """
    idx = led_num - 1  # 0-based
    is_even = idx % 2 == 0  # 偶数索引 = 奇数LED
    
    if is_even:  # LED1, LED3, LED5...
        control_bit = 3
        g_bit, r_bit, b_bit = 0, 1, 2
    else:  # LED2, LED4, LED6...
        control_bit = 7
        g_bit, r_bit, b_bit = 4, 5, 6
    
    byte = 0
    byte |= (1 if control else 0) << control_bit
    byte |= (1 if green else 0) << g_bit
    byte |= (1 if red else 0) << r_bit
    byte |= (1 if blue else 0) << b_bit
    
    # 下一个 LED 的控制位在同一字节 (奇数LED) 或下一字节 (偶数LED)
    return byte
```
